# 報告資料: テキサス・アビリーン AI キャンパス拡張と Microsoft 参画（OpenAI は追加拡張見送り）

- **調査日**: 2026-04-02
- **主出典**: [AP News](https://apnews.com/article/ai-stargate-microsoft-openai-crusoe-oracle-f4f74c3a4617d8cfab5b933fc31ccc6e)（通信社報道）／[Crusoe ニュースルーム（2026-03-27）](https://crusoe.ai/resources/newsroom/crusoe-announces-new-900-mw-ai-factory-campus-in-abilene-texas-to-support-microsoft-ai-infrastructure)（開発者・準一次）
- **要約（1文）**: 開発者 Crusoe はテキサス州アビリーンで、既存の OpenAI・Oracle 向け大規模キャンパスに隣接する **900MW 規模の新キャンパスを Microsoft 向けに新設**すると公表し、AP は **OpenAI が同地点のさらなる拡張を進めない**一方で Microsoft が隣接プロジェクトを引き受ける構図と報じた。

---

## 1. 背景 — なぜ今この情報が出ているか

- **事実（時系列・公開情報）**:
  - **2026-03-27（Crusoe 公式）**: アビリーンに **Microsoft 向けの専用「AI factory」キャンパス**を開発すると発表。**900MW**、**建物2棟＋オンサイト発電所**（系統レジリエンス用）。既存アビリーン施設と合わせサイト全体で **合計約 2.1GW** の想定容量。整地・準備は着手済み、**第1棟の通電は 2027年半ば頃**の見込み（Crusoe 記載）。
  - **既存フェーズ（Crusoe 公式）**: 第1期の **100MW×2 棟**は約1年で通電。第2期で **6 棟追加し合計 1.2GW** となる工程は **2026年末までに完了予定**と記載。各新棟は **臨界 IT 負荷 336MW** 設計など、超高密度計算向けの記述あり。
  - **Microsoft コメント（Crusoe リリース経由）**: Noelle Walsh（Cloud Operations & Innovation 社長）が、需要増に応じ **信頼性と責任ある大規模インフラ**の確保に注力している旨を述べたと掲載。
  - **AP の文脈**: 同じアビリーンで Crusoe が **OpenAI と Oracle** 向けにより大きな計算キャンパスを建設中。**Stargate** の旗艦の一つとして、トランプ前大統領の就任直後の公式発表とも結びつけて説明。Crusoe は **SoftBank も投資パートナー**と記載。
  - **OpenAI 側の公式寄りの説明（AP による引用）**: 同月早い段階で OpenAI はアビリーンの**さらなる拡張は行わない**方針を示したと AP は書く。計算インフラ責任者 Sachin Katti の **X（旧 Twitter）投稿**を引用し、「旗艦 Stargate サイトは米国最大級の一つ」「追加拡張も検討したが、**余剰容量は他地域に振り分けた**」とする旨が伝えられる。米国内で **6 か所以上**のサイトを開発中、**ウィスコンシンで Oracle と建設中の拠点**にも言及（いずれも AP 経由の要約・引用）。
  - **Microsoft と OpenAI の関係（AP）**: Microsoft はかつて OpenAI の**独占的クラウド提供者**だったが、現在は **約27%の出資**を維持しつつ AI 開発は**別軸で進める**様子が強まっている、と記述。両社が**同一地盤上で隣接**する構図である点を強調。
  - **電力・環境（AP）**: Microsoft 側の2施設に付随する新発電所は **900MW**、既存の OpenAI・Oracle 側の **350MW** ガス火力（Oracle はバックアップ電源として説明したと AP）より大きい、と記載。AI と気候コミットメントの緊張について AP は別リンクで論じている。

- **公開情報から読み取れる動機**:
  - **Crusoe / Microsoft**: 大規模 AI 需要に応える **ギガワット級の現物インフラ**を短期サイクルで供給する実績の「証明」としてのプレス。
  - **OpenAI（Katti 発言の要旨）**: **地理的分散**とポートフォリオ最適化——「アビリーンを捨てた」のではなく「**ここ以上は広げず、他に回す**」という公式ナラティブ（AP 経由）。
  - **メディア（The Register 等）**: OpenAI・Oracle が**追加キャパシティのリースを検討していたが交渉・資金調達が頓挫**した、といった説明が出ている。**当事者3社の合同プレスリリースではない**ため、因果の強さは「報道ベース」とみなすのが妥当。

- **補足・推測（根拠付き）**:
  - 同一敷地・同一開発者でも **顧客が OpenAI から Microsoft に「入れ替わった」**わけではなく、**既存 Stargate 系（OpenAI+Oracle）**と**別キャンパス（Microsoft 専用）**が隣接する二層構造、と読むのが一次資料と整合的。
  - Microsoft が当該敷地を使う動機は、**自社モデル・Copilot・Azure 需要**向けの容量確保と推測されるが、**リリースは具体的なワークロード内訳を明言していない**。

## 3. 効果 — 短期・中期・長期の予想

時間軸の定義: **短期 0〜3か月 / 中期 3か月〜1年 / 長期 1年以上**。

- **短期（0〜3か月）**:
  - **予想される効果**: 「OpenAI vs Microsoft インフラ分岐」の**ナラティブ**が強まり、テックメディアの追跡記事が増える。Crusoe・地方政治の**広報材料**として利用。
  - **不確実性**: 「交渉決裂」「Meta が最有力だった」などは**単一メディア依存**の情報があり、全体像は流動的。
  - **効果が出ない条件**: 他の大型ニュース（同業の資金調達・モデルリリース等）に注目が吸われる場合。

- **中期（3か月〜1年）**:
  - **予想される効果**: 2026年末目標の **1.2GW フェーズ完了**（Crusoe 表記）の進捗が焦点。**2027年半ば**の Microsoft 第1棟通電に向けた建設・系統接続の定点観測。
  - **不確実性**: 送配電・許認可・機器納期でスケジュールがずれる一般的リスク。
  - **効果が出ない条件**: 需要減や financing 環境の悪化でフェーズが遅延・縮小する場合（シナリオ）。

- **長期（1年以上）**:
  - **予想される効果**: **ギガワット級 on-site 発電＋データセンター**が米国 AI 供給の「標準装備」になり、**系統制約**への対応として普及する可能性（業界トレンドとしての推測）。
  - **不確実性**: 排出規制・炭素価格・再エネ比率義務など**政策**が建設・運用コストを押し上げうる。AP が触れたように **気候コミットメントとの緊張**が議会・規制の焦点になりうる。
  - **効果が出ない条件**: 原子力・系統強化など別ルートで電力が十分安価に供給され、オングリッドが再び優位に戻る場合（シナリオ）。
