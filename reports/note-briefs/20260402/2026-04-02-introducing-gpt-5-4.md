# 報告資料: GPT-5.4 公開（推論・コーディング・エージェント）

- **調査日**: 2026-04-02
- **主出典**: [OpenAI — Introducing GPT-5.4](https://openai.com/index/introducing-gpt-5-4/)（公表日: **2026-03-05**）
- **要約（1文）**: OpenAI は ChatGPT（GPT-5.4 Thinking / Pro）、API（`gpt-5.4` / `gpt-5.4-pro`）、Codex に **職業向け知識労働・コーディング・ネイティブ computer-use・大規模ツール連携**を束ねたフロンティアモデル GPT-5.4 を投入し、トークン効率と安全性（Preparedness / システムカード）・価格・レガシー退役日程まで含めて一括で説明した。

---

## 1. 背景 — なぜ今この情報が出ているか

- **事実（時系列・製品面）**:
  - **2026-03-05**: GPT-5.4 を **ChatGPT（Thinking / Pro）・API・Codex** で提供開始（**段階的ロールアウト**との記述）。
  - **位置づけ**: 「professional work」向けの **最も能力が高く効率的なフロンティア**とし、**GPT-5.3-Codex のコーディング強み**と、ツール・ソフト環境・表計算・スライド・文書タスクの横断改善を **単一の mainline 推論モデル**に統合したと説明。
  - **ChatGPT 側のUX**: GPT-5.4 Thinking は応答中に **事前の「思考プラン」**を見せ、**途中で方針変更**可能に。**深いウェブ調査**や長い推論での文脈維持を強化。
  - **API / Codex 側の技術的差分**:
    - **初の汎用モデルとしてネイティブ computer-use**（スクリーンショットに基づくマウス・キーボード操作、Playwright 等との併用）。
    - **最大約100万トークン**のコンテキスト（エージェントの長期タスク向け）。
    - **tool search**（大量ツール定義を毎リクエスト詰め込まず、必要時に定義を引く）。
    - **GPT-5.2 比で推論時のトークン使用量を大幅削減**（公式は「significantly fewer tokens」と表現）。
  - **画像入力**: `original` / `high` の解像度上限拡張（`original` は最大 **1,024万ピクセル合計**または**最大辺6000px**のいずれか低い方、等の仕様記載）。
  - **エコシステム**: 同日 **[ChatGPT for Excel アドイン](https://chatgpt.com/apps/spreadsheets/)** 公開、スプレッドシート・プレゼン用 **skills** の更新（GitHub の curated skills）など、**職場文脈での利用**を後押し。
  - **レガシー**: ChatGPT 有料ユーザー向けに **GPT-5.2 Thinking はモデルピッカーの Legacy で3か月維持**し、**2026-06-05 に退役**予定（公式記載）。

- **公開情報から読み取れる動機**:
  - **「Instant と Thinking の進化速度の差」**を言及し、ブランド上 **GPT-5.4 という番号で Codex と ChatGPT/API の選択を単純化**したい意図が読み取れる。
  - **エージェント実務**（computer-use、MCP 大量ツール、長文脈）が競争軸になっており、**レイテンシ・コスト・精度を同時に売る**必要があるタイミングと整合的。
  - 大型調達・インフラ投資（別件）と並行して、**「フロンティアの実用域」を広げる製品証拠**を出す役割（推測。資金調達記事との因果は公式が直接結びつけていない）。

- **補足・推測（根拠付き）**:
  - ベンチマーク数値は **ベンダー公表・評価条件（reasoning effort 等）依存**が大きい。GDPval では GPT-5.4 を **xhigh**、GPT-5.2 を **heavy** と揃えていない点など、**横比較の解釈には注意**（公式本文に明記）。
  - computer-use は **悪用リスクが高い能力**のため、Preparedness Framework 上 **High cyber capability** とし、ZDR 面でのブロック等を説明している（後述セーフティ）。

## 3. 効果 — 短期・中期・長期の予想

時間軸の定義: **短期 0〜3か月 / 中期 3か月〜1年 / 長期 1年以上**。

- **短期（0〜3か月）**:
  - **予想される効果**: 開発者ツール・業務SaaSが **5.4 対応をアピール**、チュートリアル・比較記事の増加。ChatGPT 内での **「計画の途中修正」UX** が一般ユーザーの期待値を上げる。
  - **不確実性**: ロールアウトが段階的なため、地域・プラン・用途で **体験差**が出る。レート制限・優先処理課金で **実効コスト**は利用パターン次第。
  - **効果が出ない条件**: 既存ワークフローが **レガシーモデル固定**のまま、検証コストを払わない場合は恩恵が限定的。

- **中期（3か月〜1年）**:
  - **予想される効果**: **computer-use 型エージェント**（RPA 代替、ブラウザオートメーション）のプロダクト化が加速。MCP / 大量ツール環境では **tool search 型パターン**が実装のデフォルトになりうる。
  - **不確実性**: セーフティ分類子の **偽陽性**（公式が言及）により、一部ユースケースでは運用摩擦。規制・契約・監査で **画面操作エージェント**が制限される業界もある。
  - **効果が出ない条件**: オンプレ・データ閉域要件が強く、クラウド推論＋computer-use が使えない場合。

- **長期（1年以上）**:
  - **予想される効果**: 「フロンティア＝職能横断の単一エージェント」という **製品哲学の固定化**が進み、OS・ブラウザ・業務アプリとの **境界再設計**（スーパーアプリ構想とも親和）が続く可能性。
  - **不確実性**: オープンモデル・各国主導AI・オンデバイス推論の進展で、**API 単価競争**と差別化軸が再配置されうる。
  - **効果が出ない条件**: ベンダーロックイン回避で **モデル抽象化**が徹底され、単一モデルの「版」への依存が薄まる場合。

## 4. 公式が示す指標・価格（出典限定）

以下は **Introducing GPT-5.4 記事内の表・本文**からの抜粋。第三者の独立再現ではない点に留意。

- **ベンチマーク（例）**: GDPval **83.0%**（GPT-5.2 は 70.9%）、SWE-Bench Pro（Public）**57.7%**、OSWorld-Verified **75.0%**、BrowseComp **82.7%**（GPT-5.4 Pro は BrowseComp **89.3%** 等）。
- **幻覚・誤り（社内セット）**: ユーザーが誤りをフラグしたプロンプト群で、**個別主張の偽率が 33% 減**、**応答全体に誤りが含まれる確率が 18% 減**（GPT-5.2 比、公式記載）。
- **API 価格（100万トークンあたり）**: `gpt-5.4` — 入力 **$2.50**、キャッシュ入力 **$0.25**、出力 **$15.00**。`gpt-5.2` は入力 **$1.75**、キャッシュ **$0.175**、出力 **$14.00**（同一記事内表）。Batch / Flex は標準の半額、Priority は2倍（公式記載）。
- **セーフティ**: GPT-5.4 を **Preparedness Framework 上 High cyber capability** として扱い、[システムカード](https://deploymentsafety.openai.com/gpt-5-4-thinking) に沿った保護。**CoT の監視可能性**に関するオープン評価（CoT controllability）も別記事で言及。

## 5. note 執筆メモ（任意）

- **フック案**: 「バージョン番号の話」ではなく、**computer-use と tool search が“汎用APIの標準装備”になった**ことで、エージェント設計の前提が変わる、という切り口。
- **検証が必要な一点**: ベンチマーク条件（reasoning effort、ブロックリスト、評価日差し替えの注記がある BrowseComp 等）。**読者向けに「ベンダー公表・条件付き」と必ず明記**。
- **免責・注意**: API 価格・提供地域・利用制限は変更されうる。**投資・医療・法務の判断は専門家に依頼**し、モデル出力は検証前提。
