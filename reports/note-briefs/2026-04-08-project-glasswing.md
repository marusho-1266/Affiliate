# 報告資料: Anthropic「Project Glasswing」と Claude Mythos Preview

- **調査日**: 2026-04-08
- **主出典**: [Anthropic — Project Glasswing](https://www.anthropic.com/glasswing)
- **要約（1文）**: Anthropic が未公開の汎用フロンティアモデル「Claude Mythos Preview」を、クラウド・セキュリティ・金融・半導体などの主要企業・団体と「防衛的」サイバー活用に限定して共有する枠組みを公表し、利用クレジット（最大1億ドル規模）と OSS セキュリティ向け寄付（400万ドル）をコミットした。

---

## 1. 背景 — なぜ今この情報が出ているか

- **事実（時系列）**:
  - **2026-04-07（公表）**: Anthropic が **Project Glasswing** を発表。ローンチパートナーとして **Amazon Web Services、Anthropic、Apple、Broadcom、Cisco、CrowdStrike、Google、JPMorganChase、Linux Foundation、Microsoft、NVIDIA、Palo Alto Networks** を列挙。（出典: [Glasswing ページ](https://www.anthropic.com/glasswing)）
  - 同ページにおいて、**Claude Mythos Preview** は「未公開（unreleased）の汎用フロンティアモデル」と位置づけられ、パートナーは同モデルを **防御的セキュリティ業務**に利用すると説明されている。
  - **過去数週間**に Mythos Preview で **数千件のゼロデイ相当**（開発者未認知の欠陥）を発見したうえ、主要OS・主要ブラウザ等にも及ぶと記載。例として **OpenBSD（27年越えの欠陥）**、**FFmpeg（16年越え）**、**Linux カーネル複数脆弱性の連鎖**など、修正済みの一部について [Frontier Red Team ブログ](https://red.anthropic.com/2026/mythos-preview) で技術詳細を公開する旨。
  - **アクセス範囲**: 上記ローンチパートナーに加え、**重要ソフトウェア基盤を構築・維持する40以上の組織**にも拡張アクセス。
  - **コミットメント（文言どおり）**: Mythos Preview 向け **利用クレジット最大1億米ドル（up to $100M）**、および **オープンソースセキュリティ組織への直接寄付400万米ドル（$4M）**。（出典: 同上）
  - **ベンチマーク（自己報告）**: 同ページで **CyberGym** の「Cybersecurity Vulnerability Reproduction」において Mythos Preview **83.1%**、Claude Opus 4.6 **66.6%** と比較表記。SWE-bench 系などコーディング評価の数値も掲載（測定条件・注記は原文参照）。
  - **リスク・整合性**: [Claude Mythos Preview — Alignment and Safety Assessment](https://www.anthropic.com/claude-mythos-preview-risk-report) が別途リンクされている。

- **公開情報から読み取れる動機**:
  - 同社は、フロンティアモデルが **脆弱性発見・悪用まで到達する能力**を持ちつつあり、その拡散が **経済・公共安全・国家安全保障**に及びうる一方、**防衛側が先に同能力を運用に載せる必要**がある、という **二重利用（dual-use）と時間競争**のフレーミングを明示している。
  - 「数か月単位で能力が進む」「防衛側が先行するには今動く必要がある」といった **緊急性**の主張が繰り返されている。
  - 政府・OSS メンテナ・他フロンティア開発者など **多主体の役割**を列挙し、単独では解けない問題として **業界連合**を前面に出している。

- **補足・推測（根拠付き）**:
  - **競合・市場**: Google（Vertex 経由提供に言及）、Microsoft、AWS 等が同時に名を連ねており、**単一ベンダ独占ではない「業界標準化」への圧**や、他社の類似「防衛特化アクセス」施策との **ナラティブ競争**がある可能性はあるが、**各社の資本・契約条件は本ページからは読み取れない**（推測）。
  - **規制・政策**: ページは民主主義国・同盟の文脈で安全保障を語るが、**特定の法案・命令と直結した発表**ではない（事実としては Glasswing 本文に依存）。
  - **メディア二次報道**（Fortune 等）は要約・解釈の差がありうるため、**数値・パートナー名は一次（Anthropic）で照合**すること。

|----------|----------------------------------|-------------------------|
| **大手クラウド／セキュリティ企業** | Mythos Preview を **自社の防御パイプライン**（コードレビュー、脆弱性調査、ハードニング）に組み込み、知見を業界共有に寄与する。 | 自社がパートナー外でも、**「AIが短時間で脆弱性探索を行う前提」**での脅威モデル・パッチ優先度・レッドチーム設計を見直す材料にする。 |
| **金融・重要インフラ（例: JPMC）** | 次世代ツールを **独自基準で評価**し、協業の可否・範囲を決める（同社CISOコメントは「厳格かつ独立したアプローチ」と明記）。 | **ベンダ提供の「強力さ」主張と、自社の統制・ログ・データ持ち出し規程**を切り離して検証する視点。 |
| **OSS メンテナ・クリティカル依存ソフト** | 限定アクセスのもとで **先行修正・スキャン**に活用し、エコシステム全体の欠陥密度を下げる（Linux Foundation 声明の趣旨）。 | 利用ポリシー・CVE 開示・エンバゴの扱いを、**コミュニティの期待と整合**させる必要がある点に注意（公開情報上の一般的論点）。 |

| **一般開発者・中小企業** | 直接は Mythos に触れられない可能性が高いが、**「攻撃側も同種ツールを使う」前提**でセキュリティ投資・インシデント対応を加速する、という間接メッセージ。 | **依存ライブラリの更新、公開インターフェースの最小化、監査ログ**など、モデル非依存の基礎対策を優先。 |

## 3. 効果 — 短期・中期・長期の予想

**時間軸の定義（本資料内）**

- **短期**: 直近 **0〜3か月**
- **中期**: **3か月〜1年**
- **長期**: **1年以上**

- **短期（0〜3か月）**:
  - **予想される効果**: パートナー限定での **脆弱性発見・パッチ提案のスループット増**、メディア・投資家向けの **「防衛ファースト」ブランド形成**、他社による **類似発表や製品メッセージの追随**。
  - **不確実性**: 外部から **実害削減（インシデント件数・MTTR）を検証しにくい**（アクセスが限定されるため）。ベンチマークは **ベンダ主導の評価条件**が絡む。
  - **効果が出ない条件**: ツールが **PoC 止まり**で本番パイプラインに組み込まれない、または **法務・データ境界**で利用が縮小する場合。

- **中期（3か月〜1年）**:
  - **予想される効果**: 限定公開でも **修正済みCVE・セキュリティアドバイザリの増加**が観測される可能性（ただし因果は特定困難）。クラウド経由の **「防衛用高能力モデル」SKU** のような商習慣が固まり始める可能性（推測）。
  - **不確実性**: **悪用側**も同等能力へアクセスするルート（違法・越境・漏えい等）については **本件の公開情報では定量化不可**。
  - **効果が出ない条件**: モデル能力が **さらに上の世代に置き換わり**、パートナーが再適応コストを負う、または **国際政治**でクラウド提供形態が制約される場合。

- **長期（1年以上）**:
  - **予想される効果**: **ソフトウェア開発ライフサイクル全体**（設計時からの脅威モデル、自動修正、サプライチェーンSBOM）への **AI組み込みの常態化**、国家・業界団体による **開示ルール・責任分配**の議論が活発化する可能性（一般的推論）。
  - **不確実性**: **規制**（AI・サイバー・金融）と **訴訟**（欠陥責任、誤検知による業務妨害）が足かせになる可能性。
  - **効果が出ない条件**: 「防衛特化」と称しつつ **攻撃支援に転用**される事例が表面化し、**業界連合への信頼**が損なわれる場合（シナリオであり、本件を指す事実ではない）。
