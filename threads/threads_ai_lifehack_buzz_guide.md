## 1. 調査背景・前提

Threadsは公式APIで個別投稿データの外部取得ができないため、本調査はSNS分析記事・運用ノウハウ記事・実際のThreads投稿事例をもとにトレンドを整理したものです。

### Threadsアルゴリズムの特性（2026年時点）

- **フォロワー数に関係なく外部拡散されやすい構造**を持つ。フォロワーゼロでも「For You」枠で表示される。
- アルゴリズムの評価指標は「いいね」だけでなく、**コメント・保存・プロフィール遷移・滞在時間**が重要。
- 拡散の連鎖モデル：**コメント → 会話 → スコア上昇 → 露出増 → バズ**
- 画像より**テキスト投稿**の方が圧倒的に伸びやすい。
- 2026年注目機能：**Dear algo**（ユーザーが好みをAIに直接リクエストする機能、現在日本未提供）

## 3. バズりやすい投稿の5つの型

### 型①：驚き・数字フック型

**特徴：** バズ度が最も高い。数字の具体性が読者を止める力になる。

**構成フロー：**
```
数字で驚かせる → 何をしたか → 再現手順3〜5個 → CTA（保存促し）
```

**AIプロンプト：**
```
以下のテーマで、Threads用の投稿文を作成してください。

【構成】
1行目（フック）：「〇〇が△△分で終わった」など数字を使って驚かせる
2〜3行目：何のAIツールを使ったか・何をしたか簡潔に
4〜7行目：箇条書きで再現ステップ3〜5個
最終行：「保存して試してみて」など自然なCTA

【条件】
- 全体500文字以内
- 1行目は35文字以内で完結
- 専門用語を避けてわかりやすく
- 数字は具体的に（〇割・〇分・〇円など）

【テーマ】: （ここに業務内容を入力）
```

**投稿例：**
```
議事録作成が30分→3分になった話。

ChatGPTに会議の録音を貼るだけ。
やり方は3ステップ↓

1. 会議を音声メモで録音
2. テキスト文字起こし（Whisper）
3. 「要点と決定事項をまとめて」とChatGPTへ

これを知ってから会議後の残業がゼロになった。
保存して今日の会議から試してみて。
```

### 型②：失敗→改善 共感型

**特徴：** コメントが増えやすい。「自分もそうだった」という共感から会話が生まれる。

**構成フロー：**
```
失敗・後悔の告白 → 何が問題だったか → AIで解決した方法 → 読者への問いかけ
```

**AIプロンプト：**
```
以下のテーマで、Threads用の共感・失敗談型投稿を作成してください。

【構成】
1行目（フック）：「昔の自分に言いたい」「正直、〇〇してました」など後悔・告白形式
2〜3行目：具体的にどんな失敗・無駄があったか
4〜5行目：AIを使ったらどう変わったか（before→after）
最終行：「あなたはどうしてる？」など読者に問いかけるCTA

【条件】
- 全体400文字以内
- 上から下まで読み続けられるテンポ感
- 共感を呼ぶリアルな言葉を使う
- 上から目線にならない等身大の語り口

【テーマ】: （ここに失敗エピソードを入力）
```

**投稿例：**
```
昔の自分に言いたい。メール返信に1時間使うな、と。

「丁寧に書かなきゃ」と思って
書いては消して、書いては消して。

今はClaudeに「この状況でのメール草案を書いて」
と30秒で依頼。あとは微調整するだけ。

あなたはメール、どのくらい時間かけてる？
```

### 型③：リスト保存促し型

**特徴：** 保存率が高い。「あとで使おう」と思わせるリスト形式がフォロワー外にも拡散する。

**構成フロー：**
```
「知らないと損」フック → リスト5〜7個 → 「保存推奨」CTA
```

**AIプロンプト：**
```
以下のテーマで、Threads用の「保存される」リスト型投稿を作成してください。

【構成】
1行目（フック）：「これ知らない人、多すぎる」「仕事で使えるAIプロンプト〇選」など
2〜8行目：箇条書きリスト（各項目1〜2行、絵文字1つ付き）
最終行：「保存して使ってみてください」など柔らかいCTA

【条件】
- 全体500文字以内
- リストは5〜7個が最適
- 各項目は「〇〇するとき→こう使う」の形式が望ましい
- 初心者でも明日から使えるレベルの内容

【テーマ】: （ここにジャンルを入力　例：資料作成・メール・企画出し）
```

**投稿例：**
```
仕事で使えるAIプロンプト5選。保存推奨。

📝 議事録まとめ→「会話ログを箇条書きで要約して」
📧 メール返信→「丁寧で簡潔な返信文を作って」
💡 企画出し→「〇〇の課題を解決するアイデアを10個」
📊 資料構成→「この内容でスライド構成を提案して」
🔍 調査まとめ→「〇〇について初心者向けに説明して」

全部コピペで使えます。保存して試してみてください。
```

### 型④：問いかけ共感型

**特徴：** コメントを誘発する。「あなたは？」の問いかけがエンゲージメントを上げる。

**構成フロー：**
```
あるある問い → 共感できる状況描写 → AI解決策 → 「あなたは？」
```

**AIプロンプト：**
```
以下のテーマで、Threads用の「問いかけ＋共感」型投稿を作成してください。

【構成】
1行目（フック）：読者が「あるある」と思う問い or 状況
2〜3行目：その悩みをリアルに描写（共感ゾーン）
4〜5行目：AIを使った具体的な解決方法
最終行：「あなたはどうしてる？」「コメントで教えて」など会話を促すCTA

【条件】
- 全体350文字以内
- 親しみやすい口語調
- 読者が「自分のことだ」と感じる具体性
- 最後は必ず読者に話しかける形で終わる

【テーマ】: （ここに悩み・状況を入力）
```

**投稿例：**
```
「何から手をつければいいかわからない」
そういう朝、ありませんか。

タスクは山積みなのに
優先順位が決まらなくて
気づいたら1時間が消えてる。

そんなとき私はChatGPTに
「今日のタスク一覧から優先順位をつけて」
と丸投げするようになった。

あなたはタスク整理、どうしてる？
```

### 型⑤：before / after 変化型

**特徴：** 拡散しやすい。変化の落差が大きいほど「自分もやってみたい」という気持ちを引き出す。

**構成フロー：**
```
以前の状況（before） → AIを使い始めた → 今の状況（after） → 具体的な方法
```

**AIプロンプト：**
```
以下のテーマで、Threads用の「before/after」型投稿を作成してください。

【構成】
1〜2行目（before）：AI活用前の大変だった状況を具体的に
3行目（転換）：「〇〇を使い始めてから変わった」
4〜5行目（after）：今の状況・変化を数字や具体的事実で
6〜7行目（how）：どのAIをどう使っているか簡潔に
最終行：自然な感想 or 軽いCTA

【条件】
- 全体450文字以内
- beforeとafterの落差をできるだけ大きく見せる
- 数字（時間・回数・金額など）を必ず入れる
- 「すごい話」より「自分もできそう」と思わせるリアル感

【テーマ】: （ここに業務改善内容を入力）
```

**投稿例：**
```
去年まで、週次レポート作成に毎週3時間かけてた。

データ集めて、グラフ貼って、文章書いて。
毎週金曜の夜が憂鬱だった。

Notionと生成AIを組み合わせてから変わった。
今は30分で終わる。

やってることはシンプル。
① データをNotionに集約
② AIに「先週比と課題を200字でまとめて」と依頼
③ 微修正して提出

2時間半、毎週返ってきた。
```

## 4. 共通の黄金ルール

### 文章・構成ルール

| ルール | 詳細 |
|---|---|
| **1行目は35文字以内** | スクロールを止める唯一のチャンス。フックに全力を注ぐ |
| **数字を必ず入れる** | 「時間・割合・金額・回数」など具体的な数値が信頼と驚きを生む |
| **最後はCTAで締める** | 「保存して試してみて」「あなたはどうしてる？」など自然な言葉で |
| **専門用語を避ける** | 初心者が「自分もできそう」と感じる言葉選びが拡散を生む |
| **500文字以内を意識** | 読み切れる量が保存・コメントにつながる |

### 運用・行動ルール

| ルール | 詳細 |
|---|---|
| **投稿後1時間はアプリを開く** | 初動のコメントに3分以内で返信するとスコアが上昇する |
| **黄金時間帯に投稿する** | 朝7〜9時 / 昼12〜14時 / 夜21〜24時 |
| **コメントには「共感＋感想」で返す** | 単純な「ありがとう」より会話を続ける返し方が有効 |
| **投稿頻度より質を重視** | 毎日投稿よりも「刺さる1投稿」を狙う |
| **トピックタグは1つに絞る** | 広すぎず・狭すぎずを意識（ハッシュタグと同じ感覚で選ぶ） |

### 型の選び方早見表

| 目的 | 推奨する型 |
|---|---|
| フォロワーを増やしたい | ① 驚き・数字型 / ⑤ before/after型 |
| 保存数を増やしたい | ③ リスト保存促し型 |
| コメントを増やしたい | ② 失敗→改善型 / ④ 問いかけ共感型 |
| 信頼感を積み上げたい | ③ リスト型 + ⑤ before/after型の交互運用 |

## 6. リポジトリ運用メモ（まるしょー Threads・朝・昼）

本ガイドを **朝・昼の投稿の正本**として使う。`.cursor/skills/threads-marusho-posts/SKILL.md` と一致させ、次を既定とする。

- **型③リスト**に近い形：**1行目（テーマ＋「5つ」＋保存フック）→ ・を5つ**。**一般ユーザー**向けの言葉で、中身は **業務でAIを使うSEの現場コツ**（自慢にならない距離感）。
- **リストの直後に、締めの段落を足さない**（「保存して…」「前日の流れ…」などの追い2文は、明示指示があるときのみ）。
- **昼**はツール操作寄り、**朝**は心構え・見方寄り。**夜・noteとは連動しない**独立ネタ。note・海外IT速報の**匂わせ・誘導はしない**（製品名の連呼やニュース丸写しもしない）。
- 報道メモ型の昼が必要なときはスキルの **旧型** を参照。
