# Threads 投稿文案 — 朝・昼・夜 共通フォーマット

リポジトリ内の既存例（`*-threads-morning.md` / `*-threads-midday.md` / `*-threads-evening.md` および単発 `*-threads.md`）と `.cursor/skills/threads-marusho-posts/SKILL.md` を踏まえた**下書き用テンプレ**です。文体・立ち位置はスキル本文に従う。

---

## ファイル名（推奨）

```
YYYY-MM-DD-〈トピック英語スラッグ〉-threads(-morning|-midday|-evening).md
```

- **朝・昼・夜でファイルを分ける**場合は末尾に `-morning` / `-midday` / `-evening`。
- **昼だけ**など部分だけの日は、該当サフィックスのみ作成してよい。

---

## Markdown 骨子（どの時間帯も共通）

```markdown
# Threads 投稿文案（〈朝｜昼｜夜〉）— 〈短い見出し〉

〈任意：想定日時・前提（例: 前夜に note 公開済み）〉

関連ブリーフ: `reports/note-briefs/...`（あれば）
関連記事: `note/YYYYMMDD/....md`
速報ログ: `reports/overseas-it-news/news-log.md`（起点にした場合）

※補足メモ（昼時点ではブリーフのみ、夜分は別ファイル、など）

---

## 投稿1（メイン）

〈冒頭ラベル〉

〈本文〉

〈夜: note 公開の一言／昼: 夜に note で何をするか／朝: 昨日の note と今日のリズム〉

〈任意: 続く↓〉

---

## 投稿2（補足・注意｜任意）

補足：

〈一次と二次の切り分け・地域制限・免責に近い注意など〉

一次・準一次ソース：

〈リンク〉

---

## 1本にまとめる場合（参考）

〈投稿1＋2を1ポスト相当に圧縮した短文〉
```

## 朝（予告・前日の振り返り）

| 項目 | 内容 |
|------|------|
| **冒頭ラベル** | `朝のITメモ` |
| **目的** | 昨日の note のテーマを1行で思い出させる。**今日も昼に絞り夜に1本**のリズム宣言。 |
| **投稿1** | 昨夜／昨日の note は「〈テーマ〉」の整理だった＋読みどころ1〜2点。今日も海外ITを追う／昼にネタを絞り夜にまとめる。 |
| **任意** | `## 投稿1（短め版）` で文字数調整。`## 投稿2` で「まだ読んでいない人は一次リンクだけでも」と誘導。`## 投稿3` で朝は URL を貼らない運用の案。 |

**投稿1（メイン）の埋め字例**

```
朝のITメモ

昨夜のnoteは、〈テーマ〉の整理でした。〈読みどころ1文〉。

今日も海外ITを追う予定。昼にネタを絞って、夜にまた1本まとめます。
```

---

## 昼（調査直後・メモ口調）

| 項目 | 内容 |
|------|------|
| **冒頭ラベル** | `昼のITメモ` |
| **目的** | 調査の**過程**。記事の丸写しにせず、芯・論点・レイヤー分け。 |
| **投稿1** | いま読んでいる報道・公式の要約＋一次と報道の違い等（本文の芯）。報道が主な根拠のときは、事実の芯を「～と報じられています」「～とも書かれています」「～と伝えられています」など媒体への帰属で書き、単独断定を避ける（例: `threads/2026-04-14-anthropic-mythos-pentagon-threads-midday.md`）。公式のみを要約する場合は「公式には～とある」などで帰属すればよい。貼り付け用の本文に Markdown 太字 `**` は入れない。**末尾の締めは固定**で、「夜に、一次リンクと用語の短い注を含めて note にまとめる予定です。」の1行とする（`続く↓` の直前）。 |
| **投稿2** | `補足：` 匿名ソース・見出しの増幅注意・投資・法務に読めない断り。報道の整理では同様に「～と報じられています」等で帰属する。リンクは Markdown または裸 URL。本文に `**` は入れない。 |
| **区切り** | スレ続き用に末尾 `続く↓` を付けてもよい（1本化する場合は削除）。 |

**投稿1（メイン）の埋め字例**

```
昼のITメモ

〈媒体名〉（〈日付・掲載地など〉）では、〈誰が何をした／何が起きたか〉、と報じられています。同記事では、〈限定・不明点〉、とも書かれています。

〈一次と報道のレイヤー分け・読み分けの手がかり1〜2文〉

〈現場目線の含み・note との役割分担〉

夜に、一次リンクと用語の短い注を含めて note にまとめる予定です。

続く↓
```

## 夜（note 公開後）

| 項目 | 内容 |
|------|------|
| **冒頭ラベル** | `夜のITメモ` |
| **目的** | **【note更新】に相当する告知**、要約・所感、返信しやすい一言（任意）。 |
| **投稿1** | 本日（または今夜）**note に公開**した旨＋テーマ＋読みどころ2〜4文。`続く↓` 任意。 |
| **投稿2** | `補足だけ先に。` 注意点。一次ソース URL。`note のURLはこのスレにリプか、プロフィールから` など運用メモ。 |

**投稿1（メイン）の埋め字例**

```
夜のITメモ

今日、〈タイトル相当〉をnoteに公開しました（〈一次の根拠と日付〉）。

〈結論・読みどころ2〜4文〉

続く↓
```

## 文体・チェック（スキルと同一）

- **です・ます**。一次情報と解釈を分ける。煽り・断定の炎上誘発は避ける。
- **絵文字**は使わなくてもよい（使うなら 0〜1）。
- X と Threads で**全文コピペ一致**は避け、Threads 側に補足・問い・口語を足す。
- 投資・法務・税務の**助言に読める表現**を避ける。
- **Threads に貼る本文**（投稿1・投稿2・1本にまとめる場合のコピペ用テキスト）では、Markdown の太字記号 `**` を使わない。Threads では太字にならず記号だけが見える。
