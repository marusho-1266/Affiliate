# Threads 投稿文案 — 朝・昼・夜 共通フォーマット

リポジトリ内の既存例（`*-threads-morning.md` / `*-threads-midday.md` / `*-threads-evening.md` および単発 `*-threads.md`）と `.cursor/skills/threads-marusho-posts/SKILL.md` を踏まえた**下書き用テンプレ**です。**朝・昼の既定**は `threads/threads_ai_lifehack_buzz_guide.md` 優先の**リスト型**（下記）。旧型の報道メモはスキル「旧型」を参照。

---

## ファイル名（推奨）

```
YYYY-MM-DD-〈トピック英語スラッグ〉-threads(-morning|-midday|-evening).md
```

- **朝・昼・夜でファイルを分ける**場合は末尾に `-morning` / `-midday` / `-evening`。
- **昼だけ**など部分だけの日は、該当サフィックスのみ作成してよい。

---

```markdown

〈任意：想定日時・前提（例: 前夜に note 公開済み）〉

関連ブリーフ: `reports/note-briefs/...`（あれば）

※補足メモ（昼時点ではブリーフのみ、夜分は別ファイル、など）
```

## 昼（調査直後・報道メモ・旧型・任意）

ユーザーが **調査メモ・報道整理** と明示するとき。`昼のITメモ`、報道帰属、「夜に、一次リンクと用語の短い注を含めて note にまとめる予定です。」で締める。例: `threads/2026-04-14-anthropic-mythos-pentagon-threads-midday.md`。

## 夜（note 公開後）

| 項目 | 内容 |
|------|------|
| **冒頭ラベル** | `夜のITメモ` |
| **目的** | **【note更新】に相当する告知**、要約・所感、返信しやすい一言（任意）。 |
| **投稿1** | 本日（または今夜）**note に公開**した旨＋テーマ＋読みどころ2〜4文。`続く↓` 任意。 |
| **投稿2** | `補足だけ先に。` 注意点。一次ソース URL。`note のURLはこのスレにリプか、プロフィールから` など運用メモ。 |

**投稿1（メイン）の埋め字例**

```
夜のITメモ

今日、〈タイトル相当〉をnoteに公開しました（〈一次の根拠と日付〉）。

〈結論・読みどころ2〜4文〉

続く↓
```

## 文体・チェック（スキルと同一）

- 朝・昼のリスト型では **短く読めること**を優先。夜・旧型では **です・ます**でもよい。一次情報と解釈を分ける。煽り・断定の炎上誘発は避ける。
- **絵文字**はリスト型では項目ごとに任意（ガイド例に合わせる）。全体ラベルに 0〜1 でもよい。
- X と Threads で**全文コピペ一致**は避け、Threads 側に補足・問い・口語を足す。
- 投資・法務・税務の**助言に読める表現**を避ける。
- **Threads に貼る本文**（投稿1・投稿2・1本にまとめる場合のコピペ用テキスト）では、Markdown の太字記号 `**` を使わない。Threads では太字にならず記号だけが見える。
