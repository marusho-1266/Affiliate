# 「ITニュース」｜Claude Security公開β、Anthropicが“防御AI”をコードレビューに入れた話

## はじめに

2026年4月30日、Anthropic は Claude 公式ブログで、法人向けのコードベース脆弱性スキャン機能 **[Claude Security](https://claude.com/blog/claude-security-public-beta)** を **パブリックベータ**として公開したと発表しました。対象は現時点で **Claude Enterprise** の顧客です。

Claude Security は、企業のリポジトリを読み、脆弱性の候補を見つけ、重要度や再現の説明を出し、さらに修正案まで提示する製品です。エンジンには **Claude Opus 4.7** が使われると説明されています。以前は **Claude Code Security** と呼ばれていた機能で、今回から Enterprise 顧客に広く開放された、という位置づけです。

読みどころは二つあります。

1つ目は、これは **Mythos Preview** のような「高度な攻撃能力も論点になるモデル」の話ではなく、**法人コードを守る側に Opus 4.7 を載せる製品**だという点です。  
2つ目は、AI が脆弱性を見つける時代に、企業側の仕事が「AIに任せる」ではなく、**検出から修正までの流れをどう管理するか**に移っていく点です。

**この記事での用語（ごく短く）**

| 語 | この記事での意味 |
| --- | --- |
| **Claude Security** | Anthropic が提供する、コードベースの脆弱性スキャンと修正案提示の機能。 |

| **AppSec** | アプリケーションセキュリティ。開発プロセスの中で脆弱性を減らす活動。 |

## 1. 何が起きたのか（結論）

先に結論です。

- Anthropic は **2026年4月30日**、**Claude Security** を **Claude Enterprise 顧客向けのパブリックベータ**として公開した。
- Claude Security は、リポジトリやディレクトリを指定してスキャンし、脆弱性の説明・重要度・再現・修正案を出す。
- 公式は、これを **Project Glasswing / Mythos Preview とは別線の、防御側向け製品**として説明している。
- The Verge も同日、Anthropic が法人向けコードベーススキャン機能を展開したと報じている。

「忙しいのでまず要点だけ知りたい」という方向けに言うと、  
**Anthropic は、サイバー能力の話を“危ないモデル”だけで終わらせず、企業のコードレビューと脆弱性管理に Opus 4.7 を入れる製品として出してきた**、というニュースです。

---

## 2. 公式情報ベースの要点（3つ）

### 2-1. 対象はまず Enterprise、Team / Max はまだ具体日なし

公式ブログでは、Claude Security は **Claude Enterprise customers** 向けにパブリックベータとして利用可能になったと書かれています。製品ページの FAQ でも、現時点では **Claude Enterprise** が対象で、**Claude Team / Max** へのアクセスは coming soon とされています。

つまり、個人ユーザーや小規模チームがすぐ同じ条件で使える、という話ではありません。まずは Enterprise の管理者が、管理コンソールから有効化し、対象リポジトリやスキャン範囲を決める流れです。

この点は、記事化するときに意外と大事です。  
「Claude がコードを全部見てくれるようになった」と広く言い切るより、**企業向けプランの管理下で使う防御機能**と捉えた方が、公式情報に近い読みになります。

### 2-2. “見つける”だけでなく、修正までの流れを意識している

Claude Security は、単に「脆弱性らしきものを列挙する」だけではありません。公式ブログでは、次のような流れが説明されています。

- リポジトリ、特定ディレクトリ、ブランチなどを指定してスキャンする。
- コードの相互作用やデータフローを読み、文脈依存の問題を探す。
- 各発見について、なぜ重要か、どの程度の確信度か、どう再現できるかを説明する。
- 修正案を出し、Claude Code on the Web で文脈を見ながら修正できるようにする。

従来の静的解析ツールは、既知パターンの検出に強い一方で、誤検知や文脈依存の見落としが問題になりがちでした。Anthropic はここで、Claude Security が「パターン照合ではなく、熟練したセキュリティ研究者のようにコードを読む」と説明しています。

もちろん、これはベンダ側の説明です。第三者の横比較や、各社コードベースでの実測値までは、この発表だけでは判断できません。とはいえ、公式の力点は明確です。**発見件数の多さより、スキャンから修正までの時間を短くする**ことに置かれています。

### 2-3. 人間のレビューは外していない

製品ページの FAQ では、Claude は間違える可能性があるため、特に重要システムでは **提案されたパッチを必ずレビューしてから適用すべき**だと明記されています。

ここは、今回のニュースを読むうえでの安全弁です。  
Claude Security は「自動で脆弱性を直して終わり」の製品として読むより、**人間のセキュリティ担当・開発者がレビューする前提で、検出と修正案作成を速くする道具**として読む方が自然です。

Slack、Jira、CSV、Markdown、webhook など、既存の監査・チケット運用に載せるための機能も説明されています。つまり、Anthropic が売りたいのは単発のデモではなく、**企業の脆弱性管理プロセスに入り込むこと**だと読めます。

---

## 3. Mythos と混同しない

The Verge の記事でも触れられている通り、Claude Security は **Mythos** と混同しない方がよい題材です。

Mythos Preview は、Project Glasswing の文脈で、脆弱性の発見や悪用まで含む高いサイバー能力が論点になりやすいモデルです。一方、Claude Security は、**一般提供モデルである Opus 4.7 を、法人の防御・修正ワークフローに載せる製品**として説明されています。

この違いを分けると、Anthropic の最近の流れが見えやすくなります。

| 論点 | 注意 |
| --- | --- |
| **Mythos Preview** | 高度なサイバー能力、限定提供、Project Glasswing の文脈で語られやすい。 |
| **Opus 4.7** | 一般提供モデル。サイバー安全策や Cyber Verification Program とあわせて説明されている。 |
| **Claude Security** | Opus 4.7 を使った法人向け防御製品。コードスキャンと修正案提示が主役。 |
| **読み違い** | 「最強モデルを全員に解放」ではなく、「防御側の業務に組み込む製品」と読む方が近い。 |

前に Opus 4.7 の記事で整理した通り、Anthropic は「強いモデルをどう出すか」と「正当なセキュリティ用途をどう扱うか」を同時に説明しています。今回の Claude Security は、その流れの中で、**企業の普段のコードベースに接続する実装面の一段**と見られます。

## 4. 誰に何が効くか（実務メモ）

| あなたの立場 | 本稿からの持ち帰り |
| --- | --- |
| **Enterprise の管理者** | まず確認すべきは機能の有無より、**どのリポジトリを接続してよいか**。コード持ち込み、ログ、契約、顧客データの境界を確認したい。 |
| **AppSec / セキュリティ担当** | 導入するなら、発見を Slack や Jira に流すだけでなく、**誰が確信度を見て、誰が修正を承認するか**を決めておく。 |
| **開発チーム** | パッチ案は便利でも、**人間のレビューが前提**。特に認証・権限・決済・個人情報まわりは自動適用しない方がよい。 |
| **経営・調達** | 「AIセキュリティツールを入れた」で終わらせず、**スキャンから修正までの時間**や、既存ツールとの役割分担で効果を見る。 |
| **ニュース読者** | Mythos の危険性の話と、Claude Security の防御製品の話を同じ箱に入れない。 |

## 5. 短期・中期・長期の整理（観察レーン）

**時間軸の定義（本稿内）**

- **短期**: 0〜3か月  
- **中期**: 3か月〜1年  
- **長期**: 1年以上  

### 5-1. 短期（0〜3か月）

**予想される効果** — Claude Enterprise を使う企業で、パイロット導入や限定リポジトリでの試験運用が増えそうです。公式ブログが強調するのは、スキャンの定期実行、対象ディレクトリの指定、却下理由の記録、監査向けエクスポートです。現場では「一回スキャンして終わり」より、**継続的に見つけ、直す流れ**が焦点になります。

**不確実性** — パブリックベータなので、機能や運用は変わりえます。誤検知、見逃し、コードベースごとの相性は、公開記事だけでは判断できません。Opus 4.7 のサイバー系セーフガードと、正当な防御業務の境界も、組織によって確認が必要です。

**効果が出ない条件** — スキャン結果を見る人が足りない、パッチ案をレビューする人がいない、既存ツールと二重運用になるだけ、という場合は効果が薄くなります。

### 5-2. 中期（3か月〜1年）

**予想される効果** — Team / Max への拡大予告や、CrowdStrike、Microsoft Security、Palo Alto Networks、SentinelOne、TrendAI、Wiz などのパートナー統合が進むと、**AI がコードや脆弱性管理を読む**ことが、エンタープライズセキュリティの標準的な説明になっていく可能性があります。

**不確実性** — 競合ツールも近い機能を出す可能性があります。また、顧客監査や法務が「生成AIによるコード提案」をどう扱うかは、業種と法域で差が出ます。金融、医療、公共、受託開発などでは、コードやログを外部サービスに出す条件が厳しくなりやすいはずです。

**効果が出ない条件** — データ境界や契約条項を満たせず、対象コードを十分に接続できない場合です。強いツールでも、見られる範囲が狭ければ、見つけられる問題も限られます。

### 5-3. 長期（1年以上）

**予想される効果** — 長期では、静的解析ツール、コードレビュー、脆弱性管理、開発者向けAIエージェントの境界が少しずつ重なっていく可能性があります。Claude Security は、その一例として見られます。

**不確実性** — 攻撃側の AI 利用も進みます。防御ツールが進化しても、攻撃者側の探索能力がより速く伸びれば、全体のリスクは下がらないかもしれません。規制、訴訟、輸出管理、企業のデータ保護方針も影響します。

**効果が出ない条件** — 組織が人間のレビューを外し、パッチ案を過信する場合です。公式 FAQ が注意する通り、Claude は間違えます。長期的な信頼は、**AIに任せる範囲と人間が責任を持つ範囲**を切り分けられるかにかかります。

## 6. まとめ

今回のポイントを一言でまとめると、**Anthropic は Opus 4.7 を、サイバー防御の「研究」だけでなく、法人コードの実務フローに入れる段階へ進めた**ということです。

- **2026年4月30日公式**で、Claude Security のパブリックベータと Enterprise 向け提供が確認できる。  
- **Mythos Preview** とは別に、**Opus 4.7 を使った防御製品**として読むのが安全。  
- 導入判断では、性能の見出しよりも、**コード持ち込み、レビュー責任、既存ツールとの接続**が重要になる。

このニュースは、Anthropic の評価額や大型提携とは別の角度で、同社が企業向けに何を売ろうとしているかを示しています。派手なモデル名よりも、**日々のコードをどう安全に直すか**にAIを入れる話として見ると、実務に近い読みになります。
