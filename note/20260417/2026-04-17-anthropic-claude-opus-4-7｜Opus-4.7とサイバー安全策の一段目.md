## はじめに

2026年4月16日、Anthropic は公式ブログ **[Introducing Claude Opus 4.7](https://www.anthropic.com/news/claude-opus-4-7)** で、**Claude Opus 4.7** の **一般提供（generally available）** を発表しました。高度なソフトウェア工学・視覚・長時間タスクの改善に加え、**直前に公表した Project Glasswing／Claude Mythos Preview** の文脈と**明示的につなげ**、次の三点を同じ投稿で説明しています。

1. **Opus 4.7 は Mythos Preview より幅広い能力の総合では下** — 公式は "less broadly capable" と明記。**サイバー能力は意図的に抑えた**うえで、**まずこのモデルで安全策を試す**のが第一段だ、という位置づけ。  
2. **禁止・高リスクのサイバー用途**は、**自動検知してブロック**する安全策を載せた。  
3. **正当なセキュリティ用途**（脆弱性研究、ペネトレーション、レッドチーム等）の利用者には、新設の **Cyber Verification Program** への参加を案内する。

料金は **Opus 4.6 と同水準**（入力 **$5**／百万トークン、出力 **$25**／百万トークン）と記載されています。開発者向けの API 名は **`claude-opus-4-7`**。提供は Claude 各製品に加え、**API、Amazon Bedrock、Google Cloud Vertex AI、Microsoft Foundry** と列挙されています（**Bedrock 上の扱いは research preview** と読める記述が API ドキュメントにあるため、**契約・リージョン単位で再確認**が前提です。詳細は補助記事 §5）。

本稿の読みどころは次の2点です。

1. **Glasswing で語った「段階的に広げる」**と、**Opus 4.7 GA** が**同じストーリー線**でどう接続されているか。  
2. 公式が説明する **「自動検知ブロック」 と 「Cyber Verification Program」 の二層**の、**言えることと言えないことの切り分け**。

## 1. 公表の中身（公式で言えること）

### 1-1. Opus 4.7（2026-04-16）

**[Introducing Claude Opus 4.7](https://www.anthropic.com/news/claude-opus-4-7)**（Apr 16, 2026）の範囲で、少なくとも次が述べられています。

- **Opus 4.6 からの改善**として、高度なソフトウェア工学、**より高解像度の視覚**、長時間タスクでの一貫性・指示追従・自己検証に触れている。  
- **Mythos Preview との関係**: 「**less broadly capable than our most powerful model, Claude Mythos Preview**」と明記（=**幅広い能力の総合では Mythos の下**）。そのうえで **Opus 4.6 よりベンチマーク上は良い**と主張。  
- **Glasswing との接続**: 「**Last week we announced Project Glasswing**」とし、Mythos Preview の提供を限定したうえで、**「より能力の低いモデルで先にサイバー安全策を試す」**方針の**最初のモデルが Opus 4.7**だと説明。  
- **学習時の扱い**: サイバー関連能力について **「differentially reduce these capabilities」** する実験をした、と記載（外部検証は別問題）。  
- **安全策**: **禁止または高リスクのサイバー利用**を示す依頼を**自動検知してブロック**する、と説明。  
- **Cyber Verification Program**: 正当なセキュリティ用途で使いたい人向けに **[申請フォーム](https://claude.com/form/cyber-use-case)** へ誘導。  
- **提供・価格・API 名**: 上記はじめにのとおり。

### 1-2. Project Glasswing（前段・2026-04-07 前後）

**[Project Glasswing](https://www.anthropic.com/glasswing)** は、Mythos Preview とパートナー連携、クリティカルソフトの防御、クレジット・寄付のコミットなどを説明する**前提稿**です。Opus 4.7 投稿はこれを**「先週」**として参照しています。

## 4. 短期・中期・長期の整理（定義と併記）

**時間軸の定義（本稿内）**

- **短期**: 0〜3か月  
- **中期**: 3か月〜1年  
- **長期**: 1年以上  

**短期** — Opus 4.7 への切り替え、ツール連携の追従、Cyber Verification の**申請増**などが起きやすい、という読み方ができます。**不確実性**は、自動ブロックの**偽陽性・偽陰性**や、検証プログラムの**審査能力**は実データがないと読みにくい点。**効果が薄い条件**は、現場が **4.6 固定**のまま、または**別モデルへ評価だけ移す**こと。

**中期** — 公式の方針説明（**Mythos 級モデルの段階的な提供拡大**の示唆）に向け、安全策の知見を**ポリシーと実装に反映**していく段階、と読めます。**不確実性**は規制・訴訟・政治文脈。**効果が薄い条件**は、ブロックが**正当利用まで阻害**し、プログラムが**信頼されない**と判断されること。

**長期** — 「強いモデル＝サイバー用途は**ゲート付き**」が業界の**デフォルト設計**として並ぶプレッシャー（他社の TAC／サイバー向け派生などと**並行**して語られることがある。推測成分あり）。**不確実性**はオープンウェイト・オンプレ回帰・地政学。**効果が薄い条件**は、攻撃側の**回避技術**が先行し、ガバナンスコストだけが増えるパターン。

## 6. まとめ

- **Opus 4.7 GA**は、単体の性能アップだけでなく、**Glasswing で示した段階的検証**の**第一段**として公式が位置づけている、と投稿本文から読み取れます。  
- 公式は **「禁止・高リスクのサイバー利用を自動検知してブロック」** する安全策と、**正当なセキュリティ用途向けの Cyber Verification Program** の**二層**を案内しています（申請フォームの URL も投稿内に記載）。  
- 料金は **Opus 4.6 と同水準**（入力 **$5**／出力 **$25** 各百万トークン）、API 名は **`claude-opus-4-7`**、提供チャネルは **API・Bedrock・Vertex AI・Foundry**（Bedrock 上の扱いは research preview と読める記述が API ドキュメントにあります。詳細は補助記事 §5）。
