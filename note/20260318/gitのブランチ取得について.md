# Gitのブランチの取得について

![サムネイル](./images/thumbnail.png)

他のメンバーが作成したブランチを、自分のローカル環境にも反映させたい場合の手順をまとめます。

## 基本的な流れ

Gitのソース管理は、下図のように**リモートリポジトリ**と呼ばれる共有の保管場所があり、その中にベースとなるソースが格納されています。これを**mainブランチ**と呼びます。

![リモートリポジトリとmainブランチ](Generated_image.png)

各開発者は、このmainブランチを`clone`コマンドで自分のPCに取得し、そこから作業を進めます。

![cloneの説明](cloneの説明.png)

mainブランチの状態を保ったまま開発を進めたい場合は、開発用のブランチをローカルリポジトリに作成します。
※下図は、BさんがDevというブランチを作成した場合のイメージです。

![ブランチの説明](cloneとDevブランチ作成.png)


Bさんが作成したDevブランチをAさんのローカルにも反映させるには、まずBさんがDevブランチをリモートリポジトリへ反映（push）する必要があります。この時点で、リモートリポジトリにはmainとDevの2本のブランチが存在することになります。

その後、AさんがリモートリポジトリのDevブランチを取得することで、BさんのDevブランチの内容を確認できるようになります。

![Devブランチの取得流れ](pushと取得の流れ.png)

## 実際の操作方法

Git Bashで上記の操作を行う場合の例を示します。

### 1. リモートリポジトリをクローンする

最初に、リモートリポジトリのURLを指定して`clone`コマンドを実行し、自分のPCにリポジトリを取得します。

```bash
git clone <リモートリポジトリのURL>
```

これにより、mainブランチを含むリポジトリ全体がローカルに複製されます。 clone後は、指定したディレクトリ配下にリポジトリが作成されます。

### 2. ローカルリポジトリ内のブランチを確認する

```bash
git branch
```

この時点では、mainブランチのみが表示されます。

### 3. リモートの最新情報を取得する（推奨）

リモートで新しく作成されたブランチを確認するには、まず `git fetch` で最新の情報を取得します。

```bash
git fetch
```

### 4. リモートリポジトリを含めて確認する

```bash
git branch --all
```

リモート追跡ブランチ（`remotes/origin/〜`）として表示されます。

### 5. リモートのブランチをローカルに取得する

Devブランチを取得する場合の例です。以下のコマンドでローカルブランチを作成し、リモートとの紐づけ（upstream設定）も行えるため、その後の `pull` がスムーズになります。

```bash
git switch -c Dev origin/Dev
```

※`git branch Dev origin/Dev` でも作成できますが、upstreamの設定がないため、`pull` の際に毎回ブランチ名を指定する必要があります。

再度`git branch`で確認すると、Devブランチがローカルに作成されていることを確認できます。

### 6. 共有後の運用

取得したブランチの内容を確認し、必要に応じて更新（pull）することで、チーム内でのブランチ共有が可能になります。

## おわりに

本記事では、他のメンバーが作成したブランチをローカルに反映させる手順を紹介しました。ポイントは、ブランチ作成者の`push`→`git fetch`でリモート情報を更新→`git branch --all`での確認→`git switch -c <ブランチ名> origin/<ブランチ名>`での取得、そして日々の`pull`での更新です。チーム開発においてブランチの取得は日常的に行う操作ですので、ぜひ慣れておくとよいでしょう。最後までお読みいただきありがとうございました。
