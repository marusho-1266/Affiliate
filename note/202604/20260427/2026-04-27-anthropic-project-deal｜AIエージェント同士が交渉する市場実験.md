# 「ITニュース」｜Anthropic「Project Deal」──エージェント同士が交渉した社内マーケット実験

## はじめに

**2026年4月24日**、Anthropic は「Project Deal」という社内実験を公開しました。これは、サンフランシスコ拠点の従業員を対象に、Slack 上で**人間本人ではなく Claude のエージェント同士が売買交渉をする**分類広告型のマーケットを試したものです。

TechCrunch も翌 4月25日にこの実験を短報で取り上げていますが、本稿ではまず Anthropic 公式の [Project Deal](https://www.anthropic.com/features/project-deal) と、公式がリンクしている付録 PDF を中心に整理します。

今回の読みどころは、単に「AI がフリマをやったら面白かった」という話ではありません。むしろ、**AI が人間の代理で交渉するとき、モデルの性能差が交渉結果に影響し、人間側はその差に気づきにくいかもしれない**、という点です。

**この記事での用語（ごく短く）**

| 語 | この記事での意味 |
| --- | --- |
| **Claude** | Anthropic が提供する AI アシスタント／モデル群。 |
| **Opus / Haiku** | Claude のモデル種別。本実験では、より高性能な Opus と、軽量な Haiku の差が論点になります。 |
| **AI エージェント** | 人間の代わりに、一定の目的に沿ってやり取りや判断を進める AI。 |
| **分類広告型マーケット** | 売りたい人・買いたい人が品物や条件を出し合う掲示板型の市場。 |
| **jailbreak / prompt injection** | AI に本来避けるべき指示を通させたり、外部入力で挙動をずらしたりする攻撃・誘導。 |

※本稿は公開情報と公式資料をもとにした整理です。**投資助言、法務助言、契約判断ではありません。** Anthropic への取材は行っていません。AI を契約・購入・交渉に使う場合は、最新の利用規約、社内規程、専門家の確認に従ってください。

## 1. 何が起きたのか

Project Deal は、Anthropic の従業員が持つ中古品や欲しいものを材料にした、1週間の社内マーケット実験です。

公式記事によると、参加者は **69名**。Claude が事前にインタビュー形式で、売りたいもの、買いたいもの、価格、交渉スタイルなどを聞き取りました。そのうえで、Slack 上に4つの並列マーケットを用意し、エージェント同士が交渉しました。

実験のうち1つは、実際に物々交換まで行う「本番」の場でした。公式本文ではこの本番 run を **Run A** とし、全員が **Opus 4.5** を使ったものとして説明しています。

公式が示した主な数字は次の通りです。

- 参加者は **69名**
- リストされた品物は **500件超**
- 成立した取引は **186件**
- 取引価値の合計は **4,000ドル超**
- 将来、同様のエージェントに支払いをしてもよいと答えた参加者は **46%**

数字だけ見ると、社内イベントとしてはかなり盛り上がったように見えます。ただし大事なのは、これは一般消費者向けサービスではなく、**自社従業員・自選参加・社内 Slack 上の研究寄りパイロット**だという点です。

---

## 2. 実験で見えた3つのこと

### 2-1. AI は実物の取引を最後まで運べた

一つ目は、AI エージェントが、人間の希望を聞き取り、相手を探し、交渉し、取引成立まで持っていけたことです。

ここで重要なのは、対象が完全な仮想財ではなかったことです。Anthropic は、先行研究では仮想の商品や合成データベースを使う例が多かった一方で、今回の実験では**実在の人間と実物の中古品**を絡めた点を強調しています。

もちろん、これはそのまま「AI に商取引を任せてよい」という意味ではありません。公式本文も、脚注で**本番の商用システムにそのままデプロイすべきではない**という趣旨を明記しています。

それでも、AI が単なる検索や要約ではなく、交渉の往復を処理する未来を考えるには、かなり具体的な材料になります。

### 2-2. モデルの差が交渉結果に出た

二つ目は、使うモデルによって取引結果に差が出たことです。

公式と付録では、Opus と Haiku の違いが分析されています。たとえば、Opus は Haiku より平均で**約2.07件多く取引**しており、付録では `p = 0.001` などの統計値も示されています。

ここでの読みどころは、単に「高性能モデルの方が強い」という話にとどまりません。

もし将来、AI エージェントが実際の購買、契約、価格交渉に関わるようになるなら、**どのモデルを使えるかが交渉力の差になる**可能性があります。人間同士の市場では、情報量、経験、交渉スキルが差になります。AI が代理になる市場では、そこに**モデル品質の差**が加わるかもしれません。

### 2-3. 人間は「自分のエージェントが弱い」ことに気づきにくい

三つ目が、いちばん怖くて面白い点です。

Anthropic は、参加者の主観評価と実際の結果のズレに注目しています。特に、Haiku を使った参加者が不利な結果になっているにもかかわらず、その劣勢を十分に認識していない可能性がある、という読みが出てきます。

ただし、ここは切り取り注意です。

公式の「The findings」節では、満足度や格付けの一部で**有意差が出ない**結果も併記されています。つまり、「Haiku ユーザーは全員が完全にだまされた」といった強い言い方は行き過ぎです。

正確には、**交渉結果にはモデル差が見える一方で、人間の満足感や自己評価はその差をそのまま反映しない可能性がある**、という読み方が安全です。

## 3. なぜ今、この実験が重要なのか

背景には、AI エージェントが「画面の横で助言するもの」から、「人間の代わりに実行するもの」へ広がりつつある流れがあります。

メールを書くだけなら、最終的に人間が送信ボタンを押せます。コードを提案するだけなら、人間がレビューできます。ところが、買い物、予約、契約、交渉のような領域では、AI が一歩踏み込むほど、**誰が同意したのか、どこまで任せたのか、ログをどう残すのか**が問題になります。

Project Deal は社内の小さな市場です。金額も大きくありません。それでも、ここには将来の論点がかなり詰まっています。

- AI が価格交渉をするとき、相手に AI だと分かるべきか
- AI が損な条件で合意した場合、責任は誰にあるのか
- 高性能な AI を使える人ほど有利になるなら、それは市場として公平なのか
- 人間の承認は、どの段階で必要なのか
- jailbreak や prompt injection でエージェントを誘導されたらどうするのか

Anthropic がこの実験を出した意味は、完成したサービスの宣伝というより、**AI が商取引に入ってくる前に、どんな問題が見えるかを先に観察すること**にあります。

## 4. 誰が何を見ればよいか

| 読む人 | 見るべき点 | 実務メモ |
| --- | --- | --- |
| **企業のプロダクト担当** | AI がユーザーの代理でどこまで交渉・購入・予約するか | 「便利だから自動化」ではなく、承認点、取消、ログ、説明を設計に入れる。 |
| **法務・調達担当** | AI が契約や価格交渉に介入する場合の責任分界 | エージェントが出した合意を、誰の意思表示として扱うかを先に決める。 |
| **AI ガバナンス担当** | モデル差、jailbreak、prompt injection、監査ログ | 「どのモデルなら許可するか」だけでなく、「人間承認が必要な場面」を定義する。 |
| **研究者・開発者** | 付録 PDF の推計、モデル差、参加者の主観評価 | 統計の言い回しを確認し、SNS向けの強い要約に引っ張られない。 |
| **一般のITニュース読者** | 取引件数や金額より、代理交渉の意味 | 「AI が勝手に買う未来」ではなく、「人間がどこまで任せるか」の話として読む。 |

企業でこの話を読むなら、最初に見るべきは「うちも AI に交渉させよう」ではありません。

むしろ、今ある AI 機能の中で、すでに**価格、契約、発注、顧客対応、予約、キャンセル**に近い判断をしている部分がないかを棚卸しする方が現実的です。

## 5. 混同しやすい点

| 論点 | 注意 |
| --- | --- |
| **「Anthropic が AI フリマサービスを出した」** | 公式が出したのは社内研究実験です。一般向けサービスの開始ではありません。 |
| **「AI 同士の取引はもう安全」** | 公式は、本番にそのままデプロイすべきではないという趣旨の注意を置いています。 |
| **「Haiku ユーザーは必ず損をする」** | 実験上の傾向はありますが、母集団は自社従業員・自選参加で、市場一般にそのまま外挿できません。 |
| **「人間は何も見ていなかった」** | 人間は事前に希望や価格、交渉スタイルを入力しています。完全自律の市場一般とは分けて読む必要があります。 |
| **「満足度に差がない＝問題なし」** | 主観的満足と実際の交渉結果は別です。ここに今回の論点があります。 |
| **「AI が契約に署名できる未来が確定した」** | Project Deal は研究寄りのパイロットであり、法的な契約主体や署名の制度設計までは示していません。 |

## 6. 短期・中期・長期の整理

ここでは時間軸を次のように置きます。

- **短期**: 0〜3か月
- **中期**: 3か月〜1年
- **長期**: 1年以上

### 6-1. 短期（0〜3か月）

短期では、研究者、開発者、SNS、テックメディアでの議論が増えるはずです。特に「AI エージェント同士が交渉した」「186件の取引」「4,000ドル超」という数字は、見出しとして使いやすいものです。

不確実性は、数字だけが独り歩きすることです。今回の実験は、参加者も環境も限定されています。一般消費者市場、BtoB 調達、金融取引のような場面とは条件が違います。

効果が出ない条件は、Anthropic がこの実験をすぐ製品化せず、単発の研究紹介として終わる場合です。その場合、話題は一時的に盛り上がっても、実務の設計変更までは進みにくいでしょう。

### 6-2. 中期（3か月〜1年）

中期では、「エージェント品質の差」が議論のキーワードになる可能性があります。

同じ市場に参加していても、ある人は高性能なエージェントを使い、別の人は軽量なエージェントを使う。その差が価格や条件に出るなら、消費者保護、表示、説明責任、代理権限の設計が重要になります。

ただし、規制や業界ルールがすぐ整うとは限りません。企業側も、実際には人間承認を挟む設計を選ぶかもしれません。その場合、Project Deal のような「エージェント同士がかなり自由に交渉する」状況は、そのまま実務に広がらない可能性があります。

### 6-3. 長期（1年以上）

長期では、AI エージェントが商取引に入ること自体は、かなり現実味を帯びてくると見ています。

ただし、中心になるのは「AI が全部勝手に買ってくれる未来」ではなく、**人間が目的と制約を与え、AI が候補探しや条件交渉をし、最後に人間または組織が承認する**形かもしれません。

そのとき、モデル品質の差、ログの保存、相手方への表示、合意の取り消し、悪意ある入力への防御が、製品設計とルール作りの焦点になります。

効果が出ない条件は、エージェントの自律交渉が信頼を得られず、結局は「おすすめ表示」や「下書き作成」に留まる場合です。もう一つは、法務・規制・プラットフォーム側が強い制約を置き、AI の自律交渉が限定的な用途に閉じる場合です。

---

## 7. まとめ

今回の Project Deal を一言でまとめると、**AI エージェントが人間の代わりに交渉する未来は、便利さだけでなく、モデル格差と責任分界の問題を一緒に連れてくる**という話です。

- Anthropic は、社内 Slack 上で AI エージェント同士が売買交渉する1週間のパイロットを公開しました。
- 参加者69名、186件の取引、4,000ドル超の取引価値という数字はありますが、これは社内研究実験であり、一般向けサービスではありません。
- Opus と Haiku の差から、将来の市場では「どの AI を使えるか」が交渉力の差になる可能性があります。
- ただし、人間の満足感や自己評価は、実際の得失をそのまま反映しない可能性があります。

AI エージェントが商取引に入るとき、本当に問われるのは「どこまで自動化できるか」だけではありません。**どこで人間が承認し、どのログを残し、どの失敗を取り消せるようにするか**です。

Project Deal は小さな社内実験ですが、その問いをかなり早い段階で見せてくれた事例だと思います。
