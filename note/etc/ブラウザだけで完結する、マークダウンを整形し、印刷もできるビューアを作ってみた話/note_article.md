# 【個人開発】ブラウザだけで完結する、マークダウンを整形し、印刷もできるビューアを作ってみた話

## はじめに

ちょっとしたメモをMarkdownで書いて、キレイに印刷したい。
ChatGPTが返してきたMarkdownの回答を、そのまま整形してPDFにしたい。
議事録や手順書を、サッと整形して共有したい。

そんな日常の「ちょっとした困った」を、**ブラウザのタブひとつ**で解決するツールを公開しました。

その名も **[Markdown Viewer](https://markdownveiwer.onrender.com/)**。

インストール不要、ログイン不要、会員登録も不要。URLを開けばすぐに使えます。

---

## 作ったきっかけ

きっかけは、**同僚から飛んでくる「生のMarkdown」** でした。

最近は生成AIを日常的に使う人が増えてきて、Slackなどのチャットツールに、ChatGPTなどの回答が**Markdown形式のまま**貼り付けられて共有される場面がとても増えました。

```markdown
# 手順
## 1. 環境構築
- Node.js をインストール
- `npm install` を実行
...
```

こんな感じのテキストがそのまま流れてくるわけです。内容自体は有用なのですが、**見出しや箇条書きが記号のまま**なので、そのままでは少し読みにくい。手順書として保存したり、関係者に印刷して配ったりしたいときは、なおさらです。

そこで私がそれまでやっていたのは、

1. Slackからテキストをコピー
2. ローカルに `.md` ファイルを作成して貼り付け
3. **Obsidian や Cursor** で開いてプレビューを確認
4. 必要なら別ツールで印刷／PDF化

……という流れだったのですが、**地味に手間**でした。ファイル名を決めて、保存場所を決めて、アプリを開いて——「ちょっと見たいだけ」「ちょっと印刷したいだけ」なのに、準備だけで数ステップかかる。

「**貼り付けたら即プレビュー、そのまま印刷・PDF化までできるツール**」がないかと探してみたものの、意外としっくりくるものが見つかりませんでした。多機能すぎたり、印刷レイアウトが微妙だったり、ログインが必要だったり。

**無いなら作ってしまおう** ——ということで、このツールの制作を始めました。

設計方針はシンプルで、

- **ブラウザのタブひとつで完結**すること
- **貼り付けたら即プレビュー**されること
- **印刷／PDF化**までそのままできること
- **余計な機能は持たない**こと

この4点だけを守って作っています。

## 画面はこんな感じ

実際の画面はとてもシンプルです。ひと目で使い方がわかる、2ペイン構成になっています。

![Markdown Viewer の画面](./assets/screenshot-main.png)

### 上部：ツールバー

画面の一番上には、よく使う操作がまとまっています。

- **印刷**：現在のプレビュー内容を印刷（PDF化）します
- **エディタを隠す**：左側の入力エリアを折りたたんで、プレビューに集中できます
- **ライトモード / ダークモード**：ワンクリックでテーマを切り替え。設定はブラウザに保存されます
- **使い方**：別ウィンドウで詳しい使い方ガイドを開きます
- **ヘッダーに印刷時間を表示**：印刷時に右上へ日時を入れるかどうかを選べます
- **スクロールを連動**：チェックすると、エディタとプレビューのスクロール位置が同期します

### 左ペイン：Markdownエディタ

行番号付きのシンプルなエディタです。折り返し表示にも対応した論理行ベースの行番号で、長文でも位置を把握しやすくなっています。

- ローカルの `.md` / `.markdown` / `.txt` ファイルを**ドラッグ＆ドロップで読み込み**可能
- **右クリックメニュー**から、カーソル位置への「改ページ挿入」や「エディタを隠す」などの操作ができます
- 中央の区切りバーを**ドラッグすれば、左右の幅を自由にリサイズ**できます

### 右ペイン：プレビュー

入力したMarkdownがリアルタイムで描画されます。

- 見出し、リスト、太字、表、コードブロックなど、標準的なMarkdown記法に対応
- ` ```mermaid ` コードブロックは、その場で図として自動レンダリング（画面例では「開始 → 処理 → 終了」のフローチャートが表示されています）
- プレビュー領域にもファイルをドロップして読み込めます

### 下部：フッター

「このサイトについて」「使い方ガイド」「プライバシーポリシー」「利用規約」「お問い合わせ」への導線がまとまっています。初めての方も迷いません。

## 主な機能

### 1. リアルタイムプレビュー

左側のエディタに入力したMarkdownが、右側に即座に反映されます。タイピング中に見た目を確認できるので、書きながらレイアウトを調整できます。

### 2. ドラッグ＆ドロップでファイル読み込み

手元にある `.md` / `.markdown` / `.txt` ファイルを画面にドラッグするだけで読み込めます。左側のエディタにも、右側のプレビュー領域にもドロップ可能。UTF-8だけでなく、BOM付きUTF-8やShift_JISも自動判別します。

### 3. Mermaid図表に対応

Markdown内に ` ```mermaid ` コードブロックを書くだけで、フローチャート・シーケンス図・ガントチャートなどが描画されます。

```mermaid
flowchart LR
    A[開始] --> B[処理]
    B --> C[終了]
```

このような図も、テキストを書くだけでブラウザ上で自動レンダリング。印刷にもそのまま反映されます。

### 4. 印刷 / PDF出力

「印刷」ボタンから、**プレビュー部分だけ**を印刷できます。ブラウザの「PDFとして保存」を使えば、そのままPDF化もOK。

- フッターに「現在ページ / 総ページ数」を自動表示
- ヘッダーに印刷時刻を表示（任意でON/OFF）
- 右クリックメニューから**任意の位置に改ページ**を挿入可能
- 長い表も印刷時に自動で折り返し

### 5. ダークモード対応

OSのテーマとは独立して切り替え可能。選択した設定はブラウザに保存されるので、次回以降も維持されます。Mermaidの配色もダークテーマに合わせて自動で切り替わります。

### 6. 使い勝手を細かくチューニング

- **行番号付きエディタ**：折り返し行にも対応した、論理行ベースの行番号表示
- **パネル幅の自由リサイズ**：中央のバーをドラッグで、エディタとプレビューの比率を調整
- **エディタの折りたたみ**：プレビューに集中したいときは、左ペインをワンクリックで非表示
- **スクロール連動**：エディタとプレビューを同じ位置にスクロール同期
- **XSS対策**：DOMPurifyでHTMLをサニタイズ済み

## 設計のこだわり

### 「単一HTMLファイル」主義

このツールは **1枚の `index.html` で完結**しています。ビルド不要、フレームワーク不要。必要なライブラリはすべてCDNから読み込む方式なので、ソースコードも読みやすく、フォークして自分用にカスタマイズするのも簡単です。

### プライバシー最優先

入力したMarkdownの内容はサーバーに一切送信されません。すべての処理はブラウザ内で完結するため、社外秘の文書やプライベートなメモを扱うときも安心です。

（※アクセス解析としてGoogle Analytics、広告配信としてGoogle AdSenseを利用しています。詳細はサイトのプライバシーポリシーをご覧ください。）

### 「ちょうどいい」機能セット

高機能な統合エディタを目指すのではなく、「ブラウザで開いて、書いて、印刷する」という日常的な流れに必要な機能だけを丁寧に実装しました。余分な機能で迷わせない——それがこのツールの設計方針です。
