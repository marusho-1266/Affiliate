# Cursorで作る「書き続ける仕組み」入門｜購入者特典キットの受け取り方と使い方

このページは、有料マガジン **「Cursorで作る『書き続ける仕組み』入門」** の購入者向けに、連載内で使った Commands / Skills / ワークシート類をまとめて受け取るための案内ページです。

全5話を読み終えたあと、手元の Cursor フォルダにこのキットを移すことで、記事を「読んで終わり」にせず、自分の執筆環境でそのまま試せる状態にできます。

> **このページについて**
> note のマガジン自体にはファイルを直接添付できないため、キット一式はこの補足記事に添付します。  
> マガジン購入者の方は、このページから添付ファイルをダウンロードしてください。

---

## 1. 添付ファイルの中身

添付するキットには、次のファイル群が入っています。

```text
kit/
  README.md
  folder-structure.md
  release-checklist.md
  commands/
    weekly-report.md
    meeting-minutes.md
    reading-note-deep.md
    social-post.md
    social-thread.md
    radio-script.md
    thumbnail-copy.md
  skills/
    editorial-voice/
      SKILL.md
    output-quality-gate/
      SKILL.md
  worksheets/
    paid-note-design.md
    series-operation.md
```

それぞれの役割は、大きく4つです。

- **Commands:** Cursor のチャット欄から `/weekly-report` などで呼び出す作業手順。
- **Skills:** 複数の Command にまたがって効かせる、文体・禁止事項・品質チェックのルール。
- **Worksheets:** 有料noteの無料/有料境界や、連載運用を設計するための空欄シート。
- **Checklist:** note 公開前に、本文・有料ブロック・派生素材を確認するためのリスト。

---

## 2. まず読むファイル

ダウンロード後、最初に読むのは次の2つだけで十分です。

1. `kit/README.md`
2. `kit/folder-structure.md`

`README.md` には、どの順番でキットを使うかを書いています。  
`folder-structure.md` には、自分の執筆用フォルダにどう配置するかのサンプルを載せています。

全部を一気に入れる必要はありません。最初は、次の4つだけで始めるのがおすすめです。

- `commands/weekly-report.md`
- `commands/meeting-minutes.md`
- `skills/editorial-voice/SKILL.md`
- `skills/output-quality-gate/SKILL.md`

---

## 3. 配置イメージ

自分の執筆用フォルダに、次のような形で置きます。

```text
writing-workspace/
  .cursor/
    commands/
      weekly-report.md
      meeting-minutes.md
    skills/
      editorial-voice/
        SKILL.md
      output-quality-gate/
        SKILL.md
```

慣れてきたら、SNS投稿・ラジオ台本・サムネ文言用の Command や、有料note設計ワークシートを追加してください。

---

## 4. 有料noteを作りたい方へ

このキットの中で、有料noteに一番直接効くのは次のファイルです。

```text
kit/worksheets/paid-note-design.md
```

このワークシートでは、次の項目を埋めます。

- 無料側に残す観察の節
- 有料側に寄せる判断の節
- 筆者メモ5節
- 無料から有料へ渡す「地図の1文」
- 有料化してよいかの最終判定

第4話で扱った **「結論を人質にしない有料note」** を実際に組むためのシートです。まずは、過去に書いた記事1本を題材にして埋めてみてください。

---

## 5. 記事公開後に使うファイル

本稿を書いたあと、公開直後に使うのは次の3本です。

- `commands/social-post.md`
- `commands/social-thread.md`
- `commands/thumbnail-copy.md`

音声配信やラジオ台本まで作る場合は、次も使います。

- `commands/radio-script.md`

第5話で書いたように、目的は投稿数を無理に増やすことではありません。**本稿の論旨が新鮮なうちに、1週間分の発信素材へ展開しておく**ことです。

---

## 6. 注意点

このキットは、完全自動で記事を完成させるためのものではありません。

Cursor に任せるのは、構成の立ち上げ、ルールの適用、派生素材の下書きまでです。最後に、自分の言葉で整える作業は残ります。

むしろ、その最後の一手を残すために、手前の迷いを減らすのがこのキットの目的です。

---

## 7. 添付ファイル

このページに、以下の ZIP ファイルを添付してください。

```text
cursor-writing-pipeline-kit.zip
```

ZIP の中身は、この記事と同じフォルダにある `kit/` 一式です。

> **公開前メモ:** note に公開する際は、この節の下に ZIP ファイルを添付してください。添付後、ファイル名が `cursor-writing-pipeline-kit.zip` になっているか確認します。

---

## まとめ

このキットは、連載で扱った内容を自分の環境に移すための作業セットです。

- 第2話・第3話で作った Commands / Skills
- 第4話で使う有料note設計ワークシート
- 第5話で使う派生・運用の素材

この3つをまとめて、手元の Cursor フォルダに置けるようにしました。

まずは1本だけ Command を入れて、実際に動かしてみてください。全部を整えるのは、そのあとで大丈夫です。
