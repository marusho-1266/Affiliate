# 週報ドラフトを作る（運用版）

以下の入力をもとに、**社内共有向けの週報**を Markdown で作成してください。

## 入力

ユーザーが以下の3ブロックを、各3〜10行の箇条書きで貼り付けます。

1. 今週やったこと（事実ベース、主語なし）
2. 所感・気づき
3. 来週の予定・依頼事項

入力の境界は、この Command 呼び出し直後の `---` 以降から、次の `---` または末尾までです。

## 出力の型（見出しの順序・数字を省略しない）

1. **今週のハイライト（3点まで）**
2. **進捗（プロジェクト別）** — 箇条書き
3. **気づき・学び（任意）** — 1〜3点
4. **来週の予定・依頼事項** — 箇条書き
5. **来週のリスク・ブロッカー（任意）**

## 書き方のルール

- 主語は「私」を使わない。事実ベース（「〇〇を完了」「△△を相談」）で書く。
- 推測や曖昧なものは「要確認」と明記する。
- 入力にない数字を推定で書かない。

## 書かないこと

- 「今後も頑張ります」のような定型の締め
- 絵文字
- 総括コメント（「全体として良い週でした」など）

## 仕上げ

最後に **抜け漏れチェックリスト**（3項目まで）を付ける。
