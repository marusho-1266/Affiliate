# Cursorで作る「書き続ける仕組み」入門 ④｜有料noteで「どこから有料にするか」を決める技術

「note で有料記事を1本出してみたい。でも、どこから有料にすれば読者を裏切らないのか分からない」という方へ。**有料で売るべきは情報量ではなく、書き手が引き受けた判断の深さ**です。**前半は無料**で、「深さ」とは何か／失敗する有料noteの典型／サブエージェントという発想／**観察と判断の見分け方**まで。**後半（有料）**で、自分の題材にそのまま使える**有料note設計ワークシート**、**節ごとの観察／判断 振り分け表**、**有料ブロック＝「筆者メモ」の5節テンプレ**（シナリオ分岐・評価軸・セルフ反証・打ち手・観測点）、そして**5レーンのサブエージェント設計**を渡します。**「詳しくは有料で」で結論を人質にしない**有料化のやり方が、この回の主題です。

> **この連載について（第4話）**
> 本記事は有料想定の連載の第4話です。**前半（1〜4章）は無料**、**後半（5章以降）は note エディタで有料ブロックに指定**する想定で書いています。第1〜3話を読んでいなくても読めるよう、冒頭で要点を再掲します。
> **読了目安:** 約15〜18分／**想定読者:** Commands と Skills が手に馴染んできて、**有料記事を1本出してみたい**段階の非開発者。

---

## 第1〜3話の要点（1段落で）

第1話で **Cursor を「資料とチャットがつながる作業場」**と捉え、第2話で **Commands**（`.cursor/commands/〈名前〉.md` に書く定型の依頼文）、第3話で **Skills**（`.cursor/skills/〈名前〉/SKILL.md` に書く横断的な判断ルール）を扱いました。ここまでで、**毎回同じ型で作業が立ち上がる土台**が整った段階です。

第4話では、その土台の上で **「深さ」を出すための道具**を扱います。サブエージェント（複数のAIに役割分担で調べてもらう仕組み）、**観察と判断の切り分け**、そして**無料／有料の境界を記事の中でどう引くか**。ここまで来ると、記事が「まとめ」から「判断の手引き」に変わります。

---

## この記事で得られるもの

- **無料パート（1〜4章）:** 「深さ」の正体、失敗する有料noteの典型、サブエージェントの考え方、観察と判断の違い、無料で引ける境界のミニ原則。
- **有料パート（5章以降）:** 自分の題材で使える**有料note設計ワークシート**、節ごとに**観察か判断かを振り分ける表**、**有料ブロックの内部骨格（5節テンプレ）**、つまずきリスト、出荷前チェック、そして深掘り用の**サブエージェント5レーン設計**。

## 前提（短く）

- Cursor のバージョン・UI は変わります。本稿は**ファイルの配置と中身**を主に扱います。
- Commands と Skills のいくつか（週報・編集方針など）を手元に持っている前提で書きます。ない方は第2話・第3話の無料パートだけでも目を通してから戻ってきてください。
- **完全自動は目指しません**。特に本話の「判断」部分は、**最後に人が引き受ける**領域です。

## 1. そもそも「深さ」とは何か

有料で読まれる記事と、無料のまとめ記事。両者の違いを、まず整理しておきます。結論から言うと、**情報量の違いではありません**。

| 浅い記事（無料で十分） | 深い記事（有料に値する） |
|----------------------|--------------------------|
| 何が起きたかをまとめてある | 「**何を判断材料にすべきか**」が整理されている |
| 事実を網羅している | 事実の **重みづけ**と **反証**が書かれている |
| 一次情報へのリンクがある | 一次情報を **どの順で読めば誤読しないか**が書かれている |
| 専門家のコメントを引用している | **自分の立場**で「現時点でこう読む」と引き受けている |

ポイントは**右側の列が全部、「書き手の判断」を含んでいる**ことです。浅い記事は**調査結果の提示**で終わりますが、深い記事は**書き手が「この順で見れば迷わない」と責任を引き受けた設計図**になっています。

AI の要約が無料で簡単に得られる時代に、**有料で読まれる記事の主戦場は「判断の深さ」**です。そしてここを作るのが、本記事で扱う**サブエージェント**と**観察／判断の切り分け**の2つです。

### 1-1. 失敗する有料noteの典型

先に、避けるべき形を書いておきます。有料noteが弱く見えるのは、文章が下手だからではなく、**無料と有料の役割分担が曖昧**だからです。

- **結論を人質にする:** 無料部分で「結局どう見るべきか」を隠し、「続きは有料で」にしてしまう。
- **有料側が情報の置き場になっている:** 事実、引用、出典リストを有料側に移してしまい、無料だけでは誤読を防げない。
- **筆者の判断が薄い:** 有料側にあるのが「詳しい説明」だけで、シナリオ分岐・比較軸・反証・打ち手がない。
- **読者が次に何を見ればいいか分からない:** 有料を読んでも、明日以降の観測点や行動が残らない。

本記事で作るのは、これらを避けるための設計図です。無料側に事実と誤読防止を残し、有料側には**筆者が順序づけた読み**を置く。これが「結論を人質にしない」有料化です。

## 2. サブエージェント — 複数のAIに役割分担で調べてもらう

### 2-1. なぜ1人のAIでは浅くなるのか

Cursor のチャット欄で1人のAIに「〇〇について深く調べて」と頼むと、**広く浅く**な回答が返ってきます。これは AI の限界というより、**1つの視点で複数の観点を同時に深掘りするのは難しい**という構造上の問題です。

たとえば「ある新サービスの発表」を調べたいとき、少なくとも次のような**視点が並列に必要**です。

- 発表元の**公式ドキュメント・プレスリリース**に何が書かれているか
- **周辺の文脈**（前後の関連発表、業界全体の流れ）
- **反論・批判・慎重論**（熱狂の中で見落とされがちな指摘）
- **類例・比較対象**（他社や過去のケースとの対置）
- **後続で観測すべきシグナル**（数週間後に分かる手がかり）

1人のAIに「全部調べて」と頼むと、**最も目立つ視点1〜2個しか拾わない**のが普通です。結果として、記事は「発表の要約＋ありきたりな反応」に落ちます。

### 2-2. 複数のAIに「レーン」を与える

そこで、**視点ごとに別々のAIを走らせる**のがサブエージェントの考え方です。Cursor では、あるAIの中から「調査を手伝うための別のAI」を並列で起動できます（実際の呼び出しはスキル側で書きます。仕組みは後述）。

大事なのは**「何人起こすか」**ではなく、**「何の役割で分けるか」**です。役割（= レーン）が曖昧なまま人数だけ増やすと、結局同じことを5回繰り返すだけになります。

### 2-3. 非開発者目線での使いどころ

「自分は開発者じゃないし、複数AIなんて大袈裟では」と思うかもしれません。ですが、**週報や読書メモですら、軽いサブエージェント化の価値はあります**。

- 週報なら「**事実レーン（今週何があったか）**」と「**振り返りレーン（何が学びか）**」を別扱いにする。
- 読書メモなら「**要約レーン**」と「**反証レーン（この主張を疑う材料はあるか）**」を別扱いにする。

分ける単位が「AIの本数」でなくても、**扱う視点を独立させる**という発想自体が、記事や文書の深さを変えます。本格的な並列起動は有料パートで触れます。

## 3. 観察と判断の切り分け

### 3-1. 2種類の文の見分け方

記事の中に出てくる文は、大雑把に**観察**と**判断**の2種類に分かれます。

| 観察（事実に近い記述） | 判断（書き手の読み・賭け） |
|-----------------------|---------------------------|
| 「〇〇社は△△日に×を発表した」 | 「〇〇社は今後、△△の方向に舵を切る可能性が高い」 |
| 「この主張の根拠は2つある」 | 「このうち根拠Bのほうが決定的だ」 |
| 「反対意見もある」 | 「反対意見Cが見落としている点が、私の読みの核だ」 |
| 「追随報道が出やすい段階」 | 「A社よりB社のほうが現実的に見える」 |

観察は、**他の人が同じ情報に当たれば似た文を書ける**もの。判断は、**書き手の立場や重みづけが入っている**もの。文を書きながら、**この文は観察か、判断か**と自問できるようになると、記事の構造が急に見やすくなります。

### 3-2. これが「無料／有料」の境界線になる

ここから一足飛びにマネタイズの話になります。**観察の文は無料で書き、判断の文を有料側に寄せる**のが、裏切りのない有料記事の作り方です。

- **無料側**: 何が起きたか、何が分かっていて何が分からないか、誤読しやすい点
- **有料側**: 書き手の重みづけ、比較の評価軸、セルフ反証、役割別の打ち手、次に見る指標

この線の引き方は、**「結論の人質をしない」**とも言い換えられます。無料の最後で「詳しくは有料で」と結論を隠さず、**無料だけで読者が誤読せず持ち帰れる状態にする**。そのうえで、**判断の深さ**を有料で売る。これが**読者に裏切りを感じさせない有料化**の要諦です。

## 4. 無料で到達できる境界設計のミニ原則

ここまでの内容は、覚え方としては**3原則**に圧縮できます。

1. **事実と帰属は無料に残す**（「誰が、いつ、どこで」は無料の責任）
2. **分からないことは無料で明言する**（「ここは検証できない」を書き切る）
3. **判断は有料に寄せ、無料は観察に徹する**（「A より B が現実的」は有料、「両者の違いは次のとおり」は無料）

この3原則だけで、**「無料の締めが結論の人質になっていないか」**は自分でも判断できるはずです。無料パートの内容はここまで。

---

## 4-末. ここまでで出来ること / 出来ないこと

無料パートの内容で、以下は到達できます。

- 自分の書いた文を **観察／判断** に分類できる。
- ミニ原則3つで、**無料の終わり方が裏切りになっていないか**自己点検できる。
- サブエージェントの**発想**は理解できる。

一方で、次の壁は残ります。

- 自分の題材で**5レーンのサブエージェントをどう設計するか**が具体化できない。
- 節ごとに観察／判断を**どう振り分けるか**、判断に迷う文が多い。
- **有料ブロック自体を何で埋めるか**（内部骨格）が分からない。

ここから先の有料パートで、**自分の1本目の有料記事を組めるところまで**引き上げます。

## 5. 有料note設計ワークシートの使い方（ここから有料想定）

ここから先は、有料ブロック想定です。購入後に持ち帰ってほしい成果物は、単なる知識ではなく、**自分の題材に使える有料noteの設計シート**です。

先に完成形を決めます。この記事の後半を使うと、次の4点が埋まります。

1. 無料側に残す観察の節。
2. 有料側に寄せる判断の節。
3. 筆者メモ5節（シナリオ分岐・評価軸・セルフ反証・打ち手・観測点）。
4. 無料末尾から有料ブロックへ渡す「地図の1文」。

つまり、後半は「サブエージェントを知るため」ではなく、**自分の1本目の有料noteを組むため**に使います。サブエージェントは、その材料を集める道具です。

### 添削の視点

自分の記事に当てはめるときは、次の3つだけ先に見てください。

- 無料側だけで「何が起きたか」「何が分からないか」「誤読しやすい点」が分かるか。
- 有料側に、筆者の重みづけや比較軸があるか。
- 有料を読んだ後、読者が「次に何を観測すればいいか」を持ち帰れるか。

この3つが揃わないなら、まだ有料化しないほうがよいです。無料記事として出すか、判断材料を追加で集めてから有料化してください。

## 6. サブエージェントの5レーン設計（有料）

題材が変わっても使える**汎用の5レーン**を提案します。これを**そのまま Skill のサブエージェント定義**に移せば、深掘りの土台ができます。

| レーン | 役割 | 何を集めるか | 誰に向いているか |
|--------|------|--------------|------------------|
| **A. 一次情報レーン** | 公式・発信者本人の言葉を拾う | プレスリリース、公式ブログ、本人発言、原典ドキュメント | 全題材で**必須**。ここを省くと記事全体の信用が崩れる |
| **B. 周辺・文脈レーン** | 時系列と背景を拾う | 直前・直後の関連発表、業界・制度の流れ、類似の過去事例 | 発表単発ではなく「流れ」を扱う記事 |
| **C. 反論・反証レーン** | 批判・慎重論・懐疑的視点を拾う | 専門家の留保コメント、反対側の公式声明、過去の失敗例 | 熱狂的な題材ほど重要度が上がる |
| **D. 比較・類例レーン** | 対置する対象を拾う | 他社の類似製品、他制度、他国の先行例 | 「どう読むか」を立体的にする記事 |
| **E. 後続観測レーン** | 数週〜数か月後に確かめられる手がかりを拾う | 決算、規制、続報、公開指標、追加発表の予告 | 「判断の深さ」を継続的価値に変えたい記事 |

### 起動の仕組み（簡素化）

Cursor では、`Skill` の中で複数の**別AI**を並列起動できます（内部的には Task という仕組み）。第3話で作った Skill に、次のような節を追加すると機能します。

```markdown
## サブエージェントの起動

以下のレーンをそれぞれ別AIで並列に起動し、結果を結合する。

- **A. 一次情報レーン:** 「〈題材〉の公式発表・一次情報を網羅して集める」
- **B. 周辺・文脈レーン:** 「〈題材〉の前後30日の関連発表を時系列で拾う」
- **C. 反論・反証レーン:** 「〈題材〉に対する慎重論・批判・過去の類似失敗例を挙げる」
- **D. 比較・類例レーン:** 「〈題材〉と比較すべき他社・他制度・先行事例を2〜3件」
- **E. 後続観測レーン:** 「〈題材〉の続報を数週間〜3か月で確認できる観測点を挙げる」

各レーンの出力を、次の見出しで統合する: 事実／留保／反論／比較／観測点。
```

**コツ:** 各レーンの依頼文は**独立して成立する短さ**にする。1レーンに2つの質問を詰めると、結局浅くなります。また、**「一次情報のみ／二次情報を含む」**のような条件を明示すると、レーン間の役割分担が崩れにくくなります。

### 非開発者向けの現実的な使い方

最初から5レーン全部を常時起動する必要はありません。**A（一次情報）は必須、C（反論）を足すと急に記事の厚みが変わる**、というのが経験則です。B・D・E は題材に応じて足し引きしてください。**週報や読書メモなら、Aと内省系の簡易レーン2つで十分**です。

## 7. 節ごとの観察／判断 振り分け表（有料）

記事を書くとき、**節ごとに「観察」か「判断」かを決めてから書き始める**と、有料境界で迷わなくなります。汎用的な振り分け基準を表にしました。

| 記事内の節 | 典型的な中身 | 観察／判断 | 置き場所 |
|------------|--------------|-----------|----------|
| はじめに・リード | 誰がいつ何を発表したか | **観察** | 無料 |
| 事実の整理 | 公式文言・数値・日付 | **観察** | 無料 |
| 時系列・背景 | 関連イベントの並び | **観察** | 無料 |
| 誤読しやすい点 | 「〇〇と△△は別物」の注意 | **観察**（誤読防止は無料の責任） | 無料 |
| 短期・中期・長期の **予想**と**不確実性** | 「〜が起こりうる／〜は分からない」 | **観察寄り**（一般的な見立て） | 無料 |
| **シナリオ分岐の比率感** | 「A より B が現実的」 | **判断** | 有料 |
| **比較の評価軸の選び方** | 「透明性という軸で見ると〜」 | **判断**（軸選びが主張） | 有料 |
| **自分の読みへの反証** | 「私の読みが崩れる条件は〜」 | **判断**（セルフ反証） | 有料 |
| **役割別の打ち手の優先順位** | 「情シスの一手目／経営の一手目」 | **判断** | 有料 |
| **次に見る指標** | 「3か月後にこれが見えれば〜」 | **判断**（観測の順序づけ） | 有料 |
| 主な参照（出典リスト） | リンクと日付 | **観察** | 無料 |

### 迷いやすい境界

- **「効果が出ない条件」**は観察側。「一般にこういう場合に失敗する」なら無料。「この題材では、失敗条件Xが最も効く」と順序づけたら判断＝有料。
- **「専門家のコメント引用」**は観察側。ただし「コメントAよりBのほうを重く見る」と重みをつけた瞬間に判断＝有料。
- **「他社と比較して」**は、**事実の列挙**なら観察、**評価軸を自分で選んでいる**なら判断。

**原則として、文末に「筆者は〜と見る」が付けられるなら判断、付けられないなら観察**、と覚えておくと迷いが減ります。

## 8. 有料ブロックの内部骨格（5節テンプレ・有料）

有料部分は、中身なしで「有料ですよ」と置かれると読者が離れます。**何があるかが見出しで想像できる**構成にします。汎用的な**5節テンプレ**を提案します。

本連載では、この有料ブロックを **「筆者メモ」** と呼びます。「書き手が引き受けた判断の詰め合わせ」という意味合いで、note 記事の本文（事実・観察）とは色を変えた節として独立させるのが狙いです。テンプレの**章番号「7-1〜7-5」はあくまで例**で、実際の記事では**記事内のどこに置くか**（多くの場合、記事の最終章）に応じて数字を読み替えてください。

```markdown
## 〈章番号〉. 筆者メモ（note 本番では有料ブロック想定）

**この章で筆者が引き受けること** — §1〜§〈まとめ章〉の整理を前提に、**筆者が順序づけた読み**
（シナリオの比率感・比較の評価軸・セルフ反証・役割別の打ち手・次に見る指標）を置きます。
〈当事者〉への取材は行っていません。〈投資／法務／医療 等〉の判断は専門家や社内規程に従ってください。

### 7-1. シナリオ分岐（筆者の注視順）

- **分岐A**: （例：本線の流れが概ね正しい場合）
- **分岐B**: （例：部分的に正しいが規模・関与度が小さい場合）
- **分岐C**: （例：単発で沈静化／想定外の軌道に動く場合）

筆者としては **（A / B / C のどれ）** を現時点で**注視順の1番**に置く。
**これは「確度」ではなく「どこを先に観察するか」の順番**である旨を明記する。
根拠が**一般則**中心か**固有事情**中心かを、1〜2行で併記する。

### 7-2. 比較の評価軸（筆者が選ぶ軸）

他社・他制度・類例との対置は、**軸の取り方そのものが主張**。

- **軸1**: （例：配布の透明性／価格／ガバナンス／連携の広さ など）
- **軸2**: （対置の読み）
- **軸3**: （対置の読み）

各軸で、**勝ち筋／負け筋**を1〜2行で言い切る。

### 7-3. 反論・限界（セルフ反証）

自分の読みを、筆者自身で疑う。

- **短絡の回避**: 自分の読みで陥りやすい短絡
- **依拠の限界**: 元材料・ソースへの依存度と、それが崩れたときの影響
- **更新の条件**: どんな材料が出れば見立てを変えるか

### 7-4. 役割別の打ち手の優先順位

組織依存が強いので、**自社の文脈に当てはめて読み替えてください**の一言を添える。

- **役割A**: 一手目（　）／二手目（　）
- **役割B**: 一手目（　）／二手目（　）
- **役割C**: 一手目（　）／二手目（　）

題材によって役割は差し替える（情シス／法務／経営／投資／研究者／デザイナー／PM 等）。

### 7-5. 次に見る指標（1〜2週間〜3か月）

**読者側でも確認できる粒度**で置く。筆者が分岐更新のために継続観測するもの。

- （他社の独立確認 の観点）
- （公聴会・規制対応 の観点）
- （契約公示・公開情報 の観点）
- （当事者の公式追加発信 の観点）
- （訴訟進行・判例蓄積 の観点）
- （次の GA／仕様更新／価格改定 の観点）
```

この5節が**全部埋まる題材でないと、有料ブロックは痩せます**。逆に言うと、**5節を埋められない題材は有料化しないほうが無難**です（無料の観察記事として出すほうが読者の満足度が高い）。

## 9. つまずきリスト（有料）

| つまずき | 原因 | 対処 |
|----------|------|------|
| **有料ブロックが薄い** | 観察と判断を分けずに書いている | 書く前に §6 の振り分け表で、節ごとに観察／判断のラベルを付ける |
| **「詳しくは有料で」で無料が閉じてしまう** | 結論を有料に持ち込んでいる | **「結論は無料、比率感と打ち手が有料」**に書き直す |
| **シナリオ分岐が「確度」になっている** | 1つのシナリオを「一番ありうる」と言い切っている | **「注視順」**と明記。確度は一次情報以外からは判別できない |
| **役割別の打ち手が具体性ゼロ** | 「関係者は対応を検討すべき」で終わっている | 一手目・二手目を**具体的な動作**で書く（会議で確認／契約確認／設定変更 等） |
| **次に見る指標が読者には確認できない** | 専門家しか見られない情報源ばかり | **公開情報・公式プレス・公聴会など、読者がアクセス可能なもの**に差し替える |
| **サブエージェントの出力が重複する** | レーン定義が被っている | レーンを**独立した観点**に切り直す。被ったらどちらかを廃止 |
| **取材していない題材で書いていいか迷う** | 「調査」と「取材」の違いが曖昧 | 有料ブロック冒頭に**「当事者への取材は行っていない」**を必ず明記する |

## 10. 出荷前の品質チェック（有料）

有料記事を出す前に、次を通してください。

- [ ] 無料側だけで **「何が起きたか／一次と報道の切り分け／分からないこと」**が読者に伝わる
- [ ] 無料の締めが **結論の人質**になっていない（「詳しくは有料で」で閉じていない）
- [ ] 有料ブロックが **5節**で構成され、空節が1つもない
- [ ] シナリオ分岐が **確度ではなく注視順**として書かれている
- [ ] 役割別の打ち手が **具体的な動作**まで書けている
- [ ] 次に見る指標が **読者がアクセス可能**な粒度
- [ ] 有料冒頭に **「取材の有無」と「免責」**が1行ずつある
- [ ] 有料ブロックの直前（無料末尾）に **「筆者メモへの地図」の1文**がある
- [ ] 報道・一次事実そのものは **有料に移していない**

**最重要は最初の2つ**です。ここが崩れると、記事ではなく情報商材寄りに見られます。

## 11. 自分の題材で使える空欄テンプレ（有料）

自分の1本目の有料記事の**設計用ワークシート**です。紙でもテキストでも、このまま埋めてから記事を書き始めると、無料／有料の境界で迷わなくなります。

```markdown
## 有料記事 1本分の設計ワークシート

### 0. 題材
- 題材の1行要約:
- 今なぜ書きたいか:
- 想定読者:

### 1. サブエージェントで集めるもの（5レーン）
- A. 一次情報:
- B. 周辺・文脈:
- C. 反論・反証:
- D. 比較・類例:
- E. 後続観測:

### 2. 無料側に残す節（観察）
- はじめに（誰がいつ何を発表したか）:
- 事実の整理:
- 時系列・背景:
- 誤読しやすい点:
- 短期・中期・長期の予想と不確実性（観察レーン）:

### 3. 有料側に寄せる節（判断）
- 7-1 シナリオ分岐（注視順）:
- 7-2 比較の評価軸:
- 7-3 セルフ反証:
- 7-4 役割別の打ち手:
- 7-5 次に見る指標:

### 4. 無料→有料の地図の1文
- まとめ末尾に置く文:

### 5. 免責（有料冒頭）
- 取材の有無:
- 扱わない判断領域(投資／法務／医療等):
```

このワークシートを**埋めてから執筆する**のと、**執筆しながら考える**のとでは、記事の締まりが別物になります。**慣れるまではこの順番で書く**ことをお勧めします。

---

## まとめ

- 「深さ」は情報量ではなく**判断の厚み**。
- サブエージェントは**5レーン（一次・周辺・反証・比較・観測）**で独立させると効く。
- **観察は無料／判断は有料**、が裏切りのない境界設計の原則。
- 有料ブロックは**5節テンプレ（シナリオ・評価軸・セルフ反証・打ち手・観測点）**を骨格にすると痩せない。
- **空欄ワークシート**を埋めてから書くと、境界で迷わない。

次回（第5話・最終話）は、**本稿で作った1本の記事を「派生」で広げる方法**を扱います。ラジオ台本・SNS投稿・サムネイル、そして**題材が尽きない導線**（候補ログ・鮮度ポリシー・重複排除）、最後に**連載を腐らせない更新の回し方**まで。ここまで揃えば、**個人でも連載を止めずに走らせる仕組み**が完成します。
