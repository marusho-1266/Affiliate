# Agent Windowを1週間使ってみた感想

Cursor の Agent Window は、**2026年4月2日** に公開された **Cursor 3.0** の変更のひとつとして公式に案内されています（[Changelog「New Cursor Interface」（3.0）](https://cursor.com/changelog/3-0)、告知 [Meet the new Cursor](https://cursor.com/blog/cursor-3)）。日本語の changelog は [こちら](https://cursor.com/ja/changelog/3-0)。本稿はこの機能をおよそ1週間メインで触ってみたメモで、ニュースではなく個人の体験談として書いています。

---

## はじめに

**Agent Window** は、従来のエディタ一体型のチャットとは別に開ける、エージェント（AI との対話とツール実行）に寄せた専用ウィンドウです。公式の説明では、**ローカルリポジトリ・Git ワークツリー・クラウド・リモート SSH** など複数の環境にまたがって、**複数のエージェントを並列に走らせられる**ことが強調されています（[Changelog 3.0](https://cursor.com/changelog/3-0)）。

起動はコマンドパレット（macOS では `Cmd+Shift+P`、Windows では `Ctrl+Shift+P`）から「Agents Window」を選ぶ方法が案内されています。IDE に戻るだけの使い方も、**エディタと Agent Window を同時に開いたまま**使うこともできます。同じアップデートでは、ウィンドウ内でブラウザ上の UI に注釈を付けて指示できる **Design Mode** や、エディタ側で複数チャットを並べて見る **Agent Tabs**、`/worktree` や `/best-of-n` といったコマンドもセットで触れられます。

本稿では主に「記事下書きや AI との壁打ちを、IDE から切り離した画面で並列に進めたい」という用途で試した範囲の話に絞ります。

---

## 良かった点

### ワークスペース単位の作業が一括でできる

おそらく今回の目玉がこれなのでしょう。

複数のワークスペースをまたいだ作業が、単一のウィンドウ内で完結しているので、複数Cursorのウィンドウを立ち上げながら行っていた作業をいちいち切替せずに行えるので、作業が進めやすかったです。

AIからの回答が返ってきたチャットは、アイコンが青く光っているので、それも直観的に分かりやすくて好印象でした。

### MCP サーバーの状況が確認しやすい

自前でRedmine用のMCPサーバーを作成しているのですが、たまに接続が切れている事があります。IDEの場合、「Cursor Setting」を開いてちゃんと生存しているかを確認して…と少し面倒だったのですが、ここの確認が簡易になったの個人的に良かったです。

## 気になった点

### git 周りの挙動が少し違和感がある

環境依存の可能性はありますが、`git pull` のあと、対象ファイルに更新が入ったかのような UI 表示になることがありました。時間とともに安定していくことを期待しています。

### フォルダやファイルのドラッグ＆ドロップができない

IDE 版では、サイドバーでフォルダやファイルをマウスでドラッグして移動を直感的に行えるのですが、Agent Window では、それが出来ません。

また、チャットへのファイル添付も同様で、対象を右クリックして「Add to Chat」する必要があり、以前の操作習慣のせいで「あ、ここではできない」と何度も思い出す場面がありました。

### エディタ内ブラウザ（プレビュー）へのアクセスが分かりにくい

HTML などを右クリックから簡単に開ける操作が、こちらでは見つけられませんでした（見落としの可能性はあります）。画面上のブラウザボタンから履歴を開くことはできたので、完全に閉ざされているわけではありません。

---

## 安心した点

### Commands がそのまま使える

「当たり前」と言われればそれまでですが、ワークフロー用の Commands が問題なく動くことを確認できてホッとしました。

### チャットの内容は IDE と相互に引き継がれる

IDE で始めた会話を Agent Window で続けたり、その逆も問題なくできました。画面を分けても文脈が切れないのは、実運用ではかなり重要だと感じます。

---

## まだ試せていないこと

### Cursor Cloud を絡めた使い方

仕事で GitHub をほとんど使わないこともあり、Cloud 連携まわりは未検証のままです。機会があれば触ってみたい部分です。

---

## まとめ

- **記事作成や AI との壁打ち**は、Agent Window で並列に回すと、体感として効率が上がった印象です。
- **がっつりコードを書く開発**は、やはり IDE の方が操作の密度と慣れの面で楽だと感じました。
- 結論としては **ケースバイケースの使い分け**がよさそうです。執筆・調査・対話寄りは Agent Window、実装メインは IDE、といった棲み分けがしっくりきます。
