# 深掘り調査サブエージェント一覧

親エージェント（オーケストレーター）が **Cursor の Task ツール**で並列起動する想定の役割定義。各 `.md` の **「Task 用プロンプト」** 節をコピーし、トピック・URL・日付を埋めて渡す。

## エージェント対応表

| ID | ファイル | 担当 | 主な取得物 |
|----|----------|------|------------|
| A | [primary-official.md](primary-official.md) | 一次・公式 | アドバイザリ、CVE、政府・ベンダ公式、プレスリリース |
| B | [regulatory-policy.md](regulatory-policy.md) | 規制・政策 | 法令、施行日、当局FAQ、標準化団体 |
| C | [market-capital.md](market-capital.md) | 市場・資本 | IR、決算、投資家向け説明、大手レポート（要約） |
| D | [technical-chain.md](technical-chain.md) | 技術・サプライ | NVD、GitHub、パッチ、PoC有無、セキュリティ調査会社 |
| E | [context-narrative.md](context-narrative.md) | 文脈・履歴 | 類似過去事件、論争・訂正、年表（いずれも出典必須） |

## いつどれを起動するか

- **常に A**（事実の芯を一次に寄せる）
- **規制・政府絡み** → A + B
- **上場企業・大型調達・M&A** → A + C
- **CVE・侵害・OSS** → A + D
- **読者向けの「前史」が必要** → A + E
- **フル深掘り**（ユーザー指示 or 高リスクトピック）→ A〜E を並列（Task を複数同時起動）

## 親のマージ順（推奨）

1. **A の事実**を最優先でタイムライン化  
2. B/C/D で **補強・数値・条文** を追加（A と矛盾する場合は一次を優先し矛盾を注記）  
3. E は **「文脈」セクション**に限定し、単独では断定しない  

## readonly

調査サブエージェントは **リポジトリへの書き込み不要**（検索・読取のみ）。Task 起動時は `readonly: true` を推奨。
