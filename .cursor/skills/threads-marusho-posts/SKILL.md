# Threads 投稿（まるしょー運用）

**まるしょー**向けに、Threads 用の日本語短文を書く。X（@marusho_tech）・note（表示名まるしょー）と **同一人物**であることが分かる語り口に揃える。

## 優先順位（Threads）

| 優先 | 内容 |
|------|------|
| 正本（朝・昼） | `threads/threads_ai_lifehack_buzz_guide.md`（バズりやすい型・黄金ルール）。特に **型③リスト**を朝・昼の既定に近づける。 |
| 立ち位置（朝・昼） | **業務でAIを使うSE**の現場感に根ざしたコツを、**一般ユーザー**にもそのまま使える言葉で書く（上から目線・自慢に見えない距離感）。 |
| トーン | **一般AIユーザー**向け（仕事・学び・日常の「使い方」）。開発・コード・インフラ用語は、ユーザーが明示したときだけ。 |
| 朝・昼と夜 | **連動しない**。朝・昼はインプレッション用の独立ネタ。夜のnote内容を匂わせたり、記事誘導を挟まない（note告知は夜スロットや別チャネル）。 |
| 夜 | 【note更新】要約・所感・問い（**任意**。Threadsではリーチが期待しにくい場合は省略やX中心でもよい）。 |

## ブランドの固定要素（変えない）

| 項目 | 値 |
|------|-----|
| 呼び名 | まるしょー |
| X | @marusho_tech |
| 立ち位置 | 地方在住のSE。運用保守から転職を機に開発寄り（**Threads朝昼では表に出さなくてよい**）。 |
| 発信の軸 | 海外ITの調査・noteは **note／X 等で継続可**。Threadsの**朝・昼**はそれとは別軸で **実務AIコツ**を出す。 |
| Threads の役割（朝・昼） | 保存・共感・会話を取りにいくリスト。**ニュース本文の丸写し**にしない。**note誘導はしない**。 |

## 文体

- 短く読めることを優先。**です・ます**でも、ガイド寄りの口語でもよい。  
- 煽り・断定的な炎上誘発は避ける。note では一次と解釈を分ける（夜・記事と同じ姿勢）。  
- 絵文字は **なくてよい**。リスト型ではガイド例に合わせ **項目ごとに1個まで**も可。  
- 1ポストは **読み切りやすい長さ**（長くなるなら分割はユーザー判断）。  
- **Threads に貼る本文**（コピペ用の投稿文）では、Markdown の太字記号 `**` を使わない。Threads では太字にならず、記号がそのまま見える。

## 初投稿

```

ここでは調査のメモや、記事に書ききれなかった所感を垂れ流します。よかったら仲良くしてください。

```

## 1日のリズム（テンプレ）

### 朝（リスト型・既定）

**目的**: ガイドの **型③（リスト保存促し）** に近い形で、**心構え・見方・チェック**を5つ並べる。内容は **SEが業務でAIを使うときの勘所**を、一般ユーザーにもそのまま使える言葉に落とす。**夜・noteとは無関係なネタ**でよい。

**構成（貼り付け用はこのまま）**

```
〈1行目：テーマ＋「5つ」＋「朝イチで保存推奨」など短いフック〉

・〈項目〉
・〈項目〉
・〈項目〉
・〈項目〉
・〈項目〉
```

- **リストの直後に、締めの段落を足さない**（「前日の流れを…」「保存して…」などの追い2文は既定では書かない）。ユーザーが明示した場合のみ。

### 昼（リスト型・ツール操作寄り・既定）

**目的**: ガイドのリスト型で、**ブラウザ・プロンプト・書き出し・貼り付け**など、**手と画面に近い操作メモ**を5つ。こちらも **SEの現場で効いた手順**を一般向けに噛み砕く。**夜・noteと連動しない**。

**構成（貼り付け用はこのまま）**

```
〈1行目：操作メモ＋「5つ」＋「昼休みに保存推奨」など〉

・〈項目〉
・〈項目〉
・〈項目〉
・〈項目〉
・〈項目〉
```

- **リストの直後に締め段落を足さない**（既定）。朝と重複しないよう、昼は **操作・手順**、朝は **心構え・見方** に寄せる。

### 夜（note 公開後）

**目的**: 要約・所感・**返信やすい一言**（質問や対比）。

```
【note更新】〈記事タイトルまたは短い見出し〉

〈2〜4文：結論・読みどころ・筆者の所感〉

〈任意：読者への問い1つ〉

〈noteのURL〉
```

- 初回直後は **リンクを貼らない回**を挟んでもよい（ユーザー方針に従う）。  
- X に同じ日に流す場合、**コピペ全文一致**は避け、Threads 側に **補足・問い・口語**を足す。

## 旧型（朝・昼・任意）

ユーザーが **調査メモ・報道整理・海外IT速報の昼**などと明示したときは、従来の **報道帰属＋固定締め** 型を使ってよい。手本: `threads/2026-04-14-anthropic-mythos-pentagon-threads-midday.md`。投稿1末尾の固定文: `夜に、一次リンクと用語の短い注を含めて note にまとめる予定です。`

## 生成時のチェックリスト

- [ ] まるしょー／@marusho_tech／note との **一貫性**（別人に見えないか）  
- [ ] 朝・昼で **夜・noteを匂わせていない**（「記事で」「昨夜の続き」等の誘導なし）  
- [ ] **投資・法務・税務の助言**に読めない表現（note に免責がある前提でも、断定勧誘は避ける）  
- [ ] 朝・昼が **既定のリスト型**のとき、項目が **ちょうど5つ**で、**リスト直後に締め段落を足していない**（ユーザー指示がある場合を除く）  
- [ ] 朝・昼が **旧型（報道メモ）**のとき、報道ベースなら「～と報じられています」等の帰属があるか／固定締めで終わっているか  
- [ ] Threads 貼り付け用の本文に `**` が含まれていないか
- [ ] 夜ポストに **会話のフック**があるか（任意だが推奨）
