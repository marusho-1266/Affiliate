# Threads 投稿（まるしょー運用）

**まるしょー**向けに、Threads 用の日本語短文を書く。X（@marusho_tech）・note（表示名まるしょー）と **同一人物**であることが分かる語り口に揃える。

## ブランドの固定要素（変えない）

| 項目 | 値 |
|------|-----|
| 呼び名 | まるしょー |
| X | @marusho_tech |
| 立ち位置 | 地方在住のSE。運用保守から転職を機に開発寄り。 |
| 発信の軸 | 海外ITの調査 → 昼に絞る → **夜に note を1本** |
| Threads の役割 | 調査メモ・所感・記事に書ききれないこと。**速報だけ**にしない |

## 文体

- **です・ます**。堅すぎず、読み手と会話している感じ。  
- 煽り・断定的な炎上誘発は避ける。一次情報と解釈は分ける（note と同じ姿勢）。  
- 絵文字は **使わなくてよい**（使うなら0〜1個まで）。  
- 1ポストは **読み切りやすい長さ**（長くなるなら分割はユーザー判断）。  
- **Threads に貼る本文**（コピペ用の投稿文）では、Markdown の太字記号 `**` を使わない。Threads では太字にならず、記号がそのまま見える。

## 初投稿（案A・そのまま貼れる）

次を **変更せずそのまま** 使ってよい（初回専用）。

```
はじめまして、まるしょーです。地方でSEをしていて、いまは開発寄りの仕事をしています。
海外ITの動きを昼に調べて、夜にnoteで1本まとめる、という形で発信しています。
ここでは調査のメモや、記事に書ききれなかった所感を垂れ流します。よかったら仲良くしてください。
```

## 1日のリズム（テンプレ）

### 朝（予告・前日の振り返り）

**目的**: 文脈づけ。読者が「このアカウントは何をする人か」を思い出す。

```
（任意：昨日のnoteのテーマを1行）
（任意：今日も昼に調べて夜にnote、などリズムの宣言）
```

**例の骨子**

- 昨日の note は「〈テーマ〉」の整理でした。  
- 今日も海外ITを追う予定。昼にネタを絞って、夜に1本まとめます。

### 昼（調査直後・メモ口調）

**目的**: 「調査の過程」。詳細は note に残し、ここは **芯だけ・メモ感**。

報道が主な根拠のときは、事実の芯を「～と報じられています」「～とも書かれています」「～と伝えられています」など媒体への帰属で書く（単独断定を避ける）。リポジトリの手本: `threads/2026-04-14-anthropic-mythos-pentagon-threads-midday.md`。根拠が公式のみのときは「公式には～とある」などでよい。上記はいずれも本文中に `**` を入れずに書く。投稿1の最終行は、次の固定文で締める: 夜に、一次リンクと用語の短い注を含めて note にまとめる予定です。

```
〈媒体名〉（〈日付など〉）では、〈事実の芯〉、と報じられています。同記事では、〈限定・不明点〉、とも書かれています。
〈一次と報道のレイヤー分け・論点1〜2文〉
〈現場目線の含み・note との役割分担など、必要なら1〜2文〉
夜に、一次リンクと用語の短い注を含めて note にまとめる予定です。
```

- プレスと技術文書のズレ、数字の取り扱い、現場との距離感など、**記事のフック**になりそうな論点を優先。

### 夜（note 公開後）

**目的**: 要約・所感・**返信やすい一言**（質問や対比）。

```
【note更新】〈記事タイトルまたは短い見出し〉

〈2〜4文：結論・読みどころ・筆者の所感〉

〈任意：読者への問い1つ〉

〈noteのURL〉
```

- 初回直後は **リンクを貼らない回**を挟んでもよい（ユーザー方針に従う）。  
- X に同じ日に流す場合、**コピペ全文一致**は避け、Threads 側に **補足・問い・口語**を足す。

## 生成時のチェックリスト

- [ ] まるしょー／@marusho_tech／note との **一貫性**（別人に見えないか）  
- [ ] **投資・法務・税務の助言**に読めない表現（note に免責がある前提でも、断定勧誘は避ける）  
- [ ] 昼ポストが **記事の丸写し**になっていないか（メモ口調か）。報道ベースなら「～と報じられています」等の帰属があるか  
- [ ] 昼の投稿1が「夜に、一次リンクと用語の短い注を含めて note にまとめる予定です。」で締まっているか  
- [ ] Threads 貼り付け用の本文に `**` が含まれていないか
- [ ] 夜ポストに **会話のフック**があるか（任意だが推奨）

## ユーザーへの確認（必要なときだけ）

次が未定なら、出力前に短く聞く。

- その日の **note の公開タイトル**と **URL**  
- 朝ポストを出すか（**朝は省略して昼・夜のみ**でもよい）  
- リンクを **何本まで**貼るか（1本推奨）
