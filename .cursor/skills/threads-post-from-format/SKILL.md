# Threads 投稿文案（フォーマット駆動）

ユーザーが指定した**スロット**（朝・昼・夜のいずれか）に応じて、`threads/投稿文案フォーマット.md` の構造に合わせた **1ファイル分の下書き**を書く。語り口・禁止事項は **`.cursor/skills/threads-marusho-posts/SKILL.md`** に従う（本スキルでは繰り返さず参照）。

## スロットの正規化

会話から次を検出したら内部で正規化する（大文字小文字無視・前後空白除去）。

| 入力例 | 正規化後 | ファイル名サフィックス |
|--------|----------|-------------------------|
| 朝、あさ、morning、am | `morning` | `-morning` |
| 昼、ひる、midday、noon | `midday` | `-midday` |
| 夜、よる、evening、night | `evening` | `-evening` |

**pm** は英語では夜に寄るため、`昼` と明示されていない限り採用しない。曖昧ならユーザーに確認する。

## 実行前に読むファイル（順）

1. `threads/投稿文案フォーマット.md`（骨子・埋め字・朝昼夜の表）
2. `.cursor/skills/threads-marusho-posts/SKILL.md`（文体・チェックリスト）
3. ユーザーが `@` した **note 底稿** / **ブリーフ** / **速報ログ** など（あれば全文または該当節）

## 情報の取り方

- **朝**: 前夜／昨日公開の **note のテーマ・読みどころ**が分かる材料を優先。無ければユーザーが会話で渡したテーマでプレースホルダ可（`〈テーマ〉`）。
- **昼**: いま読んでいるソース（報道名・日付・事実の芯）、一次と二次のレイヤー分け。報道が主な根拠の段落では、事実の芯を「～と報じられています」「～とも書かれています」「～と伝えられています」など媒体への帰属で述べる（断定調の単文にしない）。投稿2の報道整理でも同様。手本: `threads/2026-04-14-anthropic-mythos-pentagon-threads-midday.md`。根拠が公式のみのときは「公式には～とある」など別の帰属でよく、報道見出しに寄せなくてよい。いずれの本文も `**` は使わない。投稿1（メイン）の貼り付け用本文は、`続く↓` の直前に次の1行をそのまま置いて締める: 夜に、一次リンクと用語の短い注を含めて note にまとめる予定です。（文言固定。note 底稿がまだなくても同じ表現でよい。）
- **夜**: **note 公開の事実**、タイトル相当、一次の根拠と日付、読みどころ、補足・一次 URL、note URL の扱い（リプ／プロフィール等はユーザー方針または既存 `threads/*.md` に合わせる）。

一次情報と解釈を混ぜない。不確実な点は断定しない。

## 出力 Markdown の必須構造

次の順で **1ファイル**として出力する（既存 `threads/2026-04-*.md` と揃える）。

**Threads 貼り付け用の段落**（`## 投稿1（メイン）` 以降の本文、`## 投稿2` の本文、`## 1本にまとめる場合` の短文）では、Markdown の太字 `**` を使わない（Threads では反映されない）。

```markdown
# Threads 投稿文案（〈朝｜昼｜夜〉）— 〈短い見出し〉

〈任意: 想定日時・前提〉

関連ブリーフ: `...`（なければ行ごと省略可）
関連記事: `note/...`
速報ログ: `reports/overseas-it-news/news-log.md`（使っていれば）

※補足メモ

---

## 投稿1（メイン）

〈冒頭ラベル〉

〈本文〉

〈任意: 続く↓〉（昼でスレ分けする場合推奨。朝・夜は任意）

---

## 投稿2（補足・注意｜任意）

...

---

## 1本にまとめる場合（参考）

〈1ポスト相当の短文〉
```

### 冒頭ラベル（スロット別・既定）

| スロット | 既定ラベル | 絵文字 |
|----------|------------|--------|
| morning | `朝のITメモ` | 付けてもよいが **0〜1個**。既定は**付けない**（既存稿は混在のため、ユーザーが「絵文字あり」と言った場合のみ `☀️`） |
| midday | `昼のITメモ` | 同上（`☀️` は任意） |
| evening | `夜のITメモ` | 同上（`🌙` は任意） |

### 朝の追加見出し（任意）

ユーザーが短尺・朝のみ URL なしなどを求めた場合、フォーマットに従い **`## 投稿1（短め版）`** や **`## 投稿2（任意・一次確認のすすめ）`** を追加してよい。

### 夜の投稿2の語頭

既存例に合わせ、補足から入る場合は **`補足だけ先に。`** または **`補足：`** を使い分けてよい（意味が通じれば可）。

## ファイル名・保存

ユーザーがパスを指定した場合はそれに保存。未指定のとき:

- ディレクトリ: `threads/`
- ベース名: `YYYY-MM-DD-〈英語スラッグ・ハイフン区切り〉-threads-〈morning|midday|evening〉.md`
- `YYYY-MM-DD` はユーザー指定があればそれ。**なければ**ユーザーコンテキストの「今日」（会話の authoritative date）を JST 前提で用いる。

スラッグはトピックを3〜6語程度の英小文字とハイフンで（既存 `threads/` ファイルに合わせる）。

## 品質チェック（出力直前）

`threads-marusho-posts` のチェックリストに加え、次を確認する。

- [ ] 見出し `#` がスロット（朝／昼／夜）と一致している
- [ ] `## 投稿1（メイン）` と `## 1本にまとめる場合（参考）` が存在する
- [ ] 昼で長い場合、投稿1末尾に `続く↓` があるか（不要なら省略でよい）
- [ ] 昼かつ報道ベースのとき、「～と報じられています」等の帰属表現が本文に入っている（投稿1の事実の芯に少なくとも1回）
- [ ] 昼の投稿1が「夜に、一次リンクと用語の短い注を含めて note にまとめる予定です。」の1行で締まっている（`続く↓` の直前）
- [ ] 投稿1・投稿2・1本にまとめる場合の本文に `**` が含まれていない
- [ ] 夜で note URL が未確定なら、既存例と同様の**差し替え注**を書いてある

## ユーザー指示の優先

会話で「絵文字なし」「1本だけ出力」「朝は URL 貼らない」などがあれば、本スキルの既定より優先する。
