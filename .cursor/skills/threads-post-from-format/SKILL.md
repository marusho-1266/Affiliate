# Threads 投稿文案（フォーマット駆動）

ユーザーが指定した**スロット**（朝・昼・夜のいずれか）に応じて、`threads/投稿文案フォーマット.md` の構造に合わせた **1ファイル分の下書き**を書く。語り口・既定の朝昼型は **`.cursor/skills/threads-marusho-posts/SKILL.md`** と **`threads/threads_ai_lifehack_buzz_guide.md`** に従う（本スキルでは繰り返さず参照）。

## スロットの正規化

会話から次を検出したら内部で正規化する（大文字小文字無視・前後空白除去）。

| 入力例 | 正規化後 | ファイル名サフィックス |
|--------|----------|-------------------------|
| 朝、あさ、morning、am | `morning` | `-morning` |
| 昼、ひる、midday、noon | `midday` | `-midday` |
| 夜、よる、evening、night | `evening` | `-evening` |

**pm** は英語では夜に寄るため、`昼` と明示されていない限り採用しない。曖昧ならユーザーに確認する。

## 実行前に読むファイル（順）

1. `threads/投稿文案フォーマット.md`（骨子・埋め字・朝昼夜の表）
2. `threads/threads_ai_lifehack_buzz_guide.md`（朝・昼の既定がリスト型のときの型・ルール）
3. `.cursor/skills/threads-marusho-posts/SKILL.md`（文体・チェックリスト・旧型）
4. ユーザーが `@` した **note 底稿** / **ブリーフ** / **速報ログ** など（**夜**や旧型のとき。朝・昼の実務Tips単体なら不要）

## 情報の取り方

- **朝（既定）**: **リスト型**。1行目にテーマ＋「5つ」＋「朝イチで保存推奨」など。次行から **・** で5項目（心構え・見方・チェック）。内容は **業務でAIを使うSEの勘所**を一般ユーザー向けに。**夜・noteと連動しない**（記事誘導・「昨夜の続き」等なし）。**リストの直後に締め段落を足さない**（ユーザーが明示した場合のみ）。
- **昼（既定）**: **リスト型・ツール操作寄り**。1行目＋ **・** で5項目（ブラウザ・プロンプト・書き出し・貼り付けなど）。**SEの現場で効いた手順**を一般向けに。朝と役割を分け、昼は **操作・手順**。**夜・noteと連動しない**。**リストの直後に締め段落を足さない**（既定）。`**` は使わない。
- **朝・昼（旧型・任意）**: ユーザーが **調査メモ・報道整理** 等と明示したとき。報道が主な根拠なら「～と報じられています」等で帰属。投稿1末尾の固定締め: `夜に、一次リンクと用語の短い注を含めて note にまとめる予定です。` 手本: `threads/2026-04-14-anthropic-mythos-pentagon-threads-midday.md`。
- **夜**: **note 公開の事実**、タイトル相当、一次の根拠と日付、読みどころ、補足・一次 URL、note URL の扱い（リプ／プロフィール等はユーザー方針または既存 `threads/*.md` に合わせる）。

一次情報と解釈を混ぜない。不確実な点は断定しない。

## 出力 Markdown の必須構造

次の順で **1ファイル**として出力する（既存 `threads/2026-04-*.md` と揃える）。

**Threads 貼り付け用の段落**（`## 投稿1（メイン）` 以降の本文、`## 投稿2` の本文、`## 1本にまとめる場合` の短文）では、Markdown の太字 `**` を使わない（Threads では反映されない）。

```markdown
# Threads 投稿文案（〈朝｜昼｜夜〉）— 〈短い見出し〉

〈任意: 想定日時・前提〉

関連ブリーフ: `...`（なければ行ごと省略可）
関連記事: `note/...`
速報ログ: `reports/overseas-it-news/news-log.md`（使っていれば）

※補足メモ

---

## 投稿1（メイン）

〈冒頭ラベル〉

〈本文〉

〈任意: 続く↓〉（昼でスレ分けする場合推奨。朝・夜は任意）

---

## 投稿2（補足・注意｜任意）

...

---

## 1本にまとめる場合（参考）

〈1ポスト相当の短文〉
```

### 冒頭ラベル（スロット別・既定）

| スロット | 既定 | 備考 |
|----------|------|------|
| morning（リスト型） | ラベル行を**別立てしない**ことが多い。1行目が「〈テーマ〉5つ。朝イチで保存推奨。」の形 | 旧型のときは `朝のITメモ` を使ってよい |
| midday（リスト型） | 同上。1行目が「〈操作メモ〉5つ。昼休みに保存推奨。」の形 | 旧型のときは `昼のITメモ` |
| evening | `夜のITメモ` | 任意で `🌙` など **0〜1個** |

リスト型では絵文字は **項目ごとに任意**（ガイド例に合わせる）。全体の先頭ラベルに絵文字を付ける必須はない。

### 朝の追加見出し（任意）

ユーザーが短尺・朝のみ URL なしなどを求めた場合、フォーマットに従い **`## 投稿1（短め版）`** や **`## 投稿2（任意・一次確認のすすめ）`** を追加してよい。

### 夜の投稿2の語頭

既存例に合わせ、補足から入る場合は **`補足だけ先に。`** または **`補足：`** を使い分けてよい（意味が通じれば可）。

## ファイル名・保存

ユーザーがパスを指定した場合はそれに保存。未指定のとき:

- ディレクトリ: `threads/`
- ベース名: `YYYY-MM-DD-〈英語スラッグ・ハイフン区切り〉-threads-〈morning|midday|evening〉.md`
- `YYYY-MM-DD` はユーザー指定があればそれ。**なければ**ユーザーコンテキストの「今日」（会話の authoritative date）を JST 前提で用いる。

スラッグはトピックを3〜6語程度の英小文字とハイフンで（既存 `threads/` ファイルに合わせる）。

## 品質チェック（出力直前）

`threads-marusho-posts` のチェックリストに加え、次を確認する。

- [ ] 見出し `#` がスロット（朝／昼／夜）と一致している
- [ ] `## 投稿1（メイン）` と `## 1本にまとめる場合（参考）` が存在する
- [ ] 朝・昼が**既定のリスト型**のとき、**・** が **ちょうど5つ**で、**リスト直後に締め段落がない**（ユーザー指示がある場合を除く）
- [ ] 昼が**旧型（報道メモ）**のとき、報道ベースなら「～と報じられています」等の帰属があるか／固定締めで終わっているか（`続く↓` の直前）
- [ ] 昼で長い場合、投稿1末尾に `続く↓` があるか（不要なら省略でよい）
- [ ] 投稿1・投稿2・1本にまとめる場合の本文に `**` が含まれていない
- [ ] 夜で note URL が未確定なら、既存例と同様の**差し替え注**を書いてある
- [ ] 朝・昼で **note・夜ポストの匂わせ・誘導**をしていない

## ユーザー指示の優先

会話で「絵文字なし」「1本だけ出力」「朝は URL 貼らない」などがあれば、本スキルの既定より優先する。
