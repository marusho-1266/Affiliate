# note／記事からラジオ台本

ユーザーが指す **Markdown 記事**（または要点メモ）を入力とし、**読み上げ用の台本**を1ファイルにまとめる。解説トーンは落ち着いたナレーション（海外IT・セキュリティ・ビジネス整理など）を既定とし、記事のトーンに合わせて調整する。

## いつ使うか

- note 記事やブログ稿を **ポッドキャスト／Voicemaker／音声配信**向けに変換したい
- **〔転換・読み上げ用〕** のような運用ルール付き台本が欲しい
- **Markdown の装飾を読み上げに載せたくない**（TTS でノイズになる）

## 作業フロー

1. **入力を読む**  
   ユーザーが `@` で指定した `.md` を優先。メタ情報（想定読者・免責・一次ソース）を拾う。

2. **尺とパート数**  
   - 記事が短い: 本編パート3〜4＋オープニング・クロージング。  
   - 中〜長: 本編パート5前後（Anthropic 台本スタイル）。  
   - 冒頭に **想定尺**（例: 読み上げ約10〜12分）を書く。話速で前後することを一文で明記。

3. **文体**  
   - 本文は **です・ます**。箇条書きは接続詞でつないで話し言葉にする。  
   - **事実**と**解釈・所感**を分ける（「公表では〜」「筆者は〜と受け止めます」など）。  
   - **人称**: 元記事の **筆者とラジオを読み上げる人が同一** のときは **一人称（私）** でよいが、下記 **「note」非言及**・**メタ表現の回避** とあわせて破綻しないようにする。「筆者は」「記事では」「記事としては」など **第三者視点の筆者呼び**は使わない。同一人物である旨は **メタ** または **制作メモ** に一文で明記する。  
     - **第三者ナレーション**（アナウンサーが他人の記事を読む場合）のみ、従来どおり「筆者は」「記事では」でもよい。ユーザーがどちらか指定した場合はそれに従う。  
   - **放送本編では「note」に言及しない**（既定）: 元ネタが note 記事でも、**オープニング〜クロージングの読み上げ本文**に「note」「記事に書いた」などは入れない。誘導は「一次情報」「配信ページの説明文」「概要欄のリンク」などに言い換える。元ファイルパスは **メタ** にのみ書く。ユーザーが「note 名を出してよい」と明示した場合のみ例外。  
   - **メタ表現の回避**（既定）: ラジオは **解説そのもの**として聞かせる。次のような **文章メタ**は避け、内容を **直接言い切る**形に書き換える。  
     - 避ける例: 「と私は書いています」「という話にしています」「と私は整理しました」「noteに書きました」「私のnoteでの書き方です」「と私は締めくくりました」  
     - 置き換えの例: 「〜と言えます」「〜でしょう」「〜が重要です」「〜とよいでしょう」「転載する際は〜が安全です」  
     - 公式が主語の事実（「公式ブログでは〜述べています」）は **メタではなく引用の説明**として残してよい。  
   - 投資・法務・医療など該当すれば **冒頭・クロージングで免責** を1文ずつ。

4. **出力ファイルの配置**  
   元記事と同じディレクトリに、ファイル名例: `〈元記事のベース名〉｜ラジオ台本.md`。ユーザー指定があれば従う。

## 台本の構造（必須に近い順）

次のブロックを **上から順** に並べる。

| ブロック | 内容 |
|----------|------|
| タイトル行 | 1行目に「ラジオ台本｜〈テーマ〉（約◯〜◯分想定・Voicemaker等向け）」のように尺と用途を書く |
| メタ | 元ネタファイル名、想定尺、トーン、注（免責・一次確認の注意）。**筆者＝ナレーター同一**かどうか。**放送本編で note 非言及**にする場合はその旨（読み上げ対象外のルールとして） |
| 読み上げのしかた | **どの行を読まないか**を明示（見出しラベル、〔転換・読み上げ用〕の行そのもの、制作メモ見出しなど） |
| オープニング | 問いかけ1〜2文＋今日のテーマ＋免責が必要なら軽く |
| 本編パート1〜N | 各パートに **〔転換・読み上げ用〕** を置き、その**直下の段落**から読み始める転換ナレーション→本文 |
| クロージング | まとめ1段落＋一次ソースの案内＋免責＋締めのあいさつ |
| 制作メモ（非読み上げ） | 間の取り方、英語の言い方、数字の扱い、配信ページ表現への言い換えなど |
| 配信説明文（コピペ用） | **短い版**（SNS・約220字目安）、**標準版**（配信ページ・約480字目安）、**タグ**（任意・カンマ区切りでよい）。必要なら「詳細は一次ソースと概要欄」など **note を明示しない誘導**に寄せる |

パート見出しは **【本編パート1｜小見出し（約◯分）】** のように **全角【】** で書く（後述の TTS ルールに合わせる）。

## TTS・音声合成向けテキスト規則

読み上げツールに **そのまま貼れる**ことを優先する。ユーザーが「Markdown 装飾は不要」と言っていない場合でも、**既定では次を使わない**。

- `*` / `**`（強調・太字）
- `` ` ``（インラインコード）と `#` 見出し記法、`---` 区切り線
- 行頭の `-` 箇条書き（制作メモは **（1）（2）** など番号にする）

**和文と英単語の間の半角スペース**

- 英単語の**直後**に日本語が続くとき、半角スペースを**入れない**（例: `OpenAIが`、`APIで`）。  
- 日本語の**直後**に英単語が続くときも、原則スペースを**入れない**（例: `のOpenAI`、`やSaaS`）。  
- **例外**: 英語句内部のスペースは残す（例: `post money`）。誤って `postmoney` にならないよう、制作メモや本文で必要なら「英語の post money のように語間のみスペース」と注記する。

**英語がスペース区切り二語の固有名（連結ルール）**

- 人名・組織・サービス名など **スペースで区切られた英語二語** をそのまま置くと、TTS が不自然に区切ることがある。次の形式に **統一してよい**（ユーザーが別形式を指定した場合はそれに従う）。  
  - **スペースを除き**、**第二語の先頭アルファベットを大文字**にして連結する（いわゆる単語境界のキャメル風）。  
  - 例: `Mustafa Suleyman` → `MustafaSuleyman`、`Microsoft Learn` → `MicrosoftLearn`、`Microsoft Foundry` → `MicrosoftFoundry`、`MAI Playground` → `MAIPlayground`。  
  - 一般表現で「二語＋意味が単語化している」もの（例: `starting at` を価格表記として扱う）も、同じく `StartingAt` のようにしてよい。読みは制作メモで「スターツ・アット」などと補足可。  
- **三語以上**（例: `Whisper large v3`）は、リポジトリの実例に合わせて `WhisperLargeV3` のように連結するか、制作メモで読みを指定する。  
- 上記と **「英語句内部のスペースを残す」例外** がぶつかる場合は、**固有名の連結を優先**し、必要なら制作メモで読みを書く。

**その他**

- 日付は読み上げ向けに **二〇二六年三月三十一日** のような表記にしてよい（ツールが読みにくければアラビア数字に変える旨を制作メモに書く）。  
- URL は本文では長く読まず、「配信ページの説明文にリンク」「概要欄」などに言い換える（**読み上げ本編では note 名を出さない**運用と整合させる）。

## 参考にする既存例（リポジトリ内）

次のファイルは **構成・語り口の参考**として読み比べてよい。

- `note/20260401/ラジオ台本_Anthropic_ClaudeCode_ソースマップ露出.md`（Markdown 見出し版・長めの尺）  
- `note/20260402/OpenAI約1220億ドル調達を公式ベースで整理｜ラジオ台本.md`（【】見出し・TTS 向けプレーン寄せ・英和スペース処理済みの例）

ユーザーが **どちらの見出しスタイル**（`##` か【】か）を希望するか明示しない場合は、**TTS 向けは【】＋プレーン**を既定とする。

## 品質チェック（出力直前）

- [ ] 転換行の直後に、パートの内容を一言で橋渡しする文がある  
- [ ] 数字・固有名詞は記事の一次ソースと矛盾していない（推測は言い切らない）  
- [ ] `*` / `` ` `` / 不要な `#` がナレーション本文に残っていない  
- [ ] 英単語＋日本語の妙な半角スペースが残っていない（英語句の内部は維持）  
- [ ] **筆者＝ナレーター同一**のとき、本文に「筆者は」「記事では」など第三者呼びが残っていない  
- [ ] **放送本編**に「note」や「記事に書いた」など、元記事の **媒体名・執筆メタ**が入っていない（メタ・制作メモ除く。ユーザーが明示した場合は例外）  
- [ ] **メタ表現**（「と私は書いています」「という話にしています」「noteに書きました」等）が読み上げ本文に残っていない。解説として言い切れている  
- [ ] **英語二語の固有名**を連結した場合、表記が台本内で一貫している（制作メモにルール例を1行）  
- [ ] 配信説明文に、免責・前提・関連記事への誘導が必要なら含める  

## ユーザー指示の優先

「尺を5分に」「二人会話に」「投資の話はしない」など、**そのセッション内の明示指示**は本スキルより優先する。
