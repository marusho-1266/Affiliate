# note 三層パック（本編無料＋筆者メモ・有料／補助無料）

「難しい」への手当てとして、**本編**（密度の高い整理）を薄めずに、**補助（無料）**で語彙・制度・読み順を足す。**筆者メモ（有料）**は「難易度」ではなく**深度・時間・判断材料**で説明できるように切り出す。

## 三層の役割

| 層 | 想定の置き場（note） | リポジトリ `note/` での典型 |
|----|----------------------|------------------------------|
| **補助（無料）** | 先に読む足場 | `…｜補助｜…md` または `…｜補助資料｜…md` |
| **本編（無料）** | 要約・論点・一次と報道の切り分けまで完結 | `…｜〈キャッチ〉.md` |
| **筆者メモ（有料想定）** | 含意・比較・反論・シナリオ・現場打ち手の優先順位など（筆者の読み・整理） | 本編で **「主な参照」等の出典リストの直後** に置き、note エディタでその節を有料化する想定。見出しの既定例: `## 筆者メモ（note 本番では有料ブロック想定）`（章番号付きでも可） |

## 含めない／含める（重なり防止）

**補助に寄せる（本編で長文化しない）**

- 用語ミニ辞典の**正本**（初出の平易定義、表で一覧）
- **おすすめ読み順**（初見／再読／筆者メモだけ、の分岐があれば一言ずつ）
- **年表・当事者一覧**（1枚で済む分量）
- **「報道と公式」レイヤー**の図式的説明（表1つ＋短い段落）

**筆者メモ（有料）に寄せる（補助だけでは代替しにくい）**

- 含意・市場・政治の**シナリオ分岐**
- 他社・他制度との**比較表**（主張が増えるもの）
- **反論・限界**の整理（筆者の判断が前面に出るもの）
- 現場の**打ち手の優先順位**（組織依存が強い具体策）

**本編（無料パート）で必ず満たすこと**

- 無料だけでも「**何が起きたか**」「**何が一次で言えて何が報道か**」「**何がまだ分からないか**」まで言える
- 補助・筆者メモがなくても**誤読しない最低限**は本編に残す（補助は足場であり、本編の必須情報を全部移さない）

## ファイル命名（リポジトリ）

- **本編:** `note/YYYYMMDD/YYYY-MM-DD-<slug>｜〈日本語キャッチ〉.md`（既存流儀に合わせる）
- **補助:** 同一フォルダ `YYYY-MM-DD-<slug>｜補助｜〈短い題〉.md` または既存例どおり `｜補助資料｜`
- **相互リンク:** 相対パス `./ファイル名.md`

## 本編の冒頭に必ず入れる文（短く）

1. **読み順:** 「初めての方は **補助 → 本編** の順がおすすめ」など**一文**
2. **補助へのリンク**（ファイル名を明記）

## 本編の「主な参照」の直後（筆者メモ・有料ブロックの目安）

- **無料パート**は本文の要約・論点・**参照・出典**まで。**筆者メモ**（有料想定）は**その下**に置く（読者が一次へ辿ったあとで筆者の読みに進む流れ）。
- 見出しの既定例: `## 筆者メモ（note 本番では有料ブロック想定）`（`## 7. 筆者メモ（…）` のように章番号を付けてもよい）
- 箇条書きで**載せる予定の論点だけ**列挙し、本文は短くてよい（note 掲載時に有料枠へ移動）
- **ブリーフ・`reports/note-briefs/`・執筆工程**は本編・補助ともに書かない（`note-article-from-brief` の禁止事項と同じ）

## 補助稿のテンプレ（見出し骨子）

```markdown
# 「ITニュース・補助」｜〈本編と対になる短い題〉

## 本稿の位置づけ
（本編へのリンク・本編が担うこと／補助が担うこと）

## おすすめの読み順
（初見／再読／有料だけ、など）

## 1. 用語ミニ辞典
## 2. 年表（必要なときだけ）
## 3. 報道と公式のレイヤー
## 4. 本編に入る前のチェックリスト（任意）
```

## 品質チェック

- [ ] 補助が「本編の焼き直し長文」になっていない（**足場**に徹している）
- [ ] 筆者メモ（有料想定）が「補助の延長」だけになっていない
- [ ] 本編冒頭に**読み順＋補助リンク**がある
- [ ] 相互リンクの相対パスが切れていない
