# note 三層パック（本編無料＋筆者メモ・有料／補助無料）

「難しい」への手当てとして、**本編**（密度の高い整理）を薄めずに、**補助（無料）**で語彙・制度・読み順を足す。**筆者メモ（有料）**は「難易度」ではなく**深度・時間・判断材料**で説明できるように切り出す。

## 三層の役割

| 層 | 想定の置き場（note） | リポジトリ `note/` での典型 |
|----|----------------------|------------------------------|
| **補助（無料）** | 先に読む足場 | `…｜補助｜…md` または `…｜補助資料｜…md` |
| **本編（無料）** | 要約・論点・一次と報道の切り分け・**時間軸の観察**まで完結 | `…｜〈キャッチ〉.md` |
| **筆者メモ（有料想定）** | 含意・比較・反論・シナリオ・現場打ち手の優先順位など（筆者の読み・整理） | 本編で **「主な参照」等の出典リストの直後** に置き、note エディタでその節を有料化する想定。見出しの既定例: `## 筆者メモ（note 本番では有料ブロック想定）`（章番号付きでも可、例 `## 7. 筆者メモ（…）`） |

## 観察 vs 判断（無料／有料の切れ目）

同じ「時間軸」「効果」「論点」の話でも、**文の強度**で置き場が変わる。**観察に近い留保**は無料、**筆者の賭け（順序づけ・比率・評価軸）**は有料、という線で揃える。

| 書き方 | 例 | 置く場所 |
|---|---|---|
| 予想される効果（一般的・観察的） | 「追随報道が出やすい段階」 | **無料**（本編） |
| 不確実性（事実に近い留保） | 「匿名ソースの範囲は外から検証できない」 | **無料**（本編） |
| 効果が出ない条件（一般的な失敗像） | 「オンボーディング摩擦で停滞」 | **無料**（本編） |
| 強い順序づけ／比率感（筆者の賭け） | 「A より B が現実的」 | **有料**（筆者メモ） |
| 他社比較の評価軸（軸選び自体が主張） | 「配布の透明性で相対的に強い」 | **有料**（筆者メモ） |
| 役割別の打ち手の優先順位（組織依存） | 「情シス一手目は認証経路の点検」 | **有料**（筆者メモ） |

## 含めない／含める（重なり防止）

**補助に寄せる（本編で長文化しない）**

- 用語ミニ辞典の**正本**（初出の平易定義、表で一覧）
- **おすすめ読み順**（初見／再読／筆者メモだけ、の分岐があれば一言ずつ）
- **年表・当事者一覧**（1枚で済む分量）
- **「報道と公式」レイヤー**の図式的説明（表1つ＋短い段落）

**本編（無料）に残す**

- 要約・論点・一次と報道の切り分け
- **短期・中期・長期の整理**（定義＋予想効果・不確実性・効果が出ない条件の**観察レーン**まで。順序づけや比率は入れない）
- **混同しやすい点のチェックリスト**（誤読防止は無料の責任）
- 主な参照（出典リスト）

**筆者メモ（有料）に寄せる（補助・無料では代替しにくい）**

- 含意・市場・政治の**シナリオ分岐**（**比率感・注視順**つき）
- 他社・他制度との**比較表**（評価軸の選び方が主張になるもの）
- **反論・限界**の整理（筆者の判断へのセルフ反証）
- 現場の**打ち手の優先順位**（**役割別の一手目・二手目**）
- **次に見る指標**（1〜2週間〜3か月で確証／否認が分かる観測点）

**本編（無料パート）で必ず満たすこと**

- 無料だけでも「**何が起きたか**」「**何が一次で言えて何が報道か**」「**何がまだ分からないか**」「**時間軸での焦点**（短期／中期／長期の観察）」まで言える
- 補助・筆者メモがなくても**誤読しない最低限**は本編に残す（補助は足場であり、本編の必須情報を全部移さない）
- **結論の人質**をしない：「詳しくは有料で」「結論は筆者メモで」のような**閉じ方は避ける**

## ファイル命名（リポジトリ）

- **本編:** `note/YYYYMMDD/YYYY-MM-DD-<slug>｜〈日本語キャッチ〉.md`（既存流儀に合わせる）
- **補助:** 同一フォルダ `YYYY-MM-DD-<slug>｜補助｜〈短い題〉.md` または既存例どおり `｜補助資料｜`
- **相互リンク:** 相対パス `./ファイル名.md`

## 本編の冒頭に必ず入れる文（短く）

1. **読み順:** 「初めての方は **補助 → 本編** の順がおすすめ」など**一文**
2. **補助へのリンク**（ファイル名を明記）

## 本編の見出し順（推奨テンプレ）

海外ニュース翻案・発表整理系では、次の順が再現性が高い（章番号は必須ではない）。

1. はじめに（リード＋読みどころ＋用語ミニ）
2. 報道で言えること／一次（公式）で言えること（節を分ける）
3. 時期文脈（なぜ今の見出しになりやすいか。必要なときだけ）
4. 混同しやすい点（チェックリスト、2列表）
5. **短期・中期・長期の整理（定義と併記）**  ←**無料の観察レーン**
6. まとめ（＋筆者メモへの地図の一文）
7. 主な参照
8. **筆者メモ（有料想定）**  ←以下 7-1〜7-5

## 筆者メモ（有料ブロック）の内部骨格

note エディタでの有料化は**この見出しから下を有料ブロックに指定**する運用。見出しの**直後**に**リード段落 1〜2 行**を置き、「何を買うのか」を無料で見える状態にするのが安全側（note の仕様でリードまでが無料で表示される前提に立てない場合は、リードを見出しの**直前・本編末尾**に置き換える）。

```markdown
## 7. 筆者メモ（note 本番では有料ブロック想定）

**この章で筆者が引き受けること** — §1〜§〈まとめ章〉の整理を前提に、**筆者が順序づけた読み**（シナリオの比率感・他社比較の評価軸・セルフ反証・役割別の打ち手・次に見る指標）を置きます。**〈当事者〉への取材は行っていません**。〈投資／法務／安全保障等〉の判断は**専門家・社内規程**に従ってください。

### 7-1. シナリオ分岐（筆者の読み・比率感つき）
- 分岐A / 分岐B / 分岐C を列挙（**確度ではなく「筆者が注視したい順」**と明示できるとより誠実）
- 現時点で筆者が現実的と見る分岐と、その根拠（一般則／固有事情の比重）

### 7-2. 比較の評価軸（筆者が選ぶ軸）
- 軸1 / 軸2 / 軸3（**軸の取り方自体が主張**）
- 他社・他制度との対置で、どの軸で勝ち筋／負け筋があるか

### 7-3. 反論・限界（セルフ反証）
- 自分の読みの短絡・依拠・一般則の限界を、筆者自身で列挙
- 見立てを**どの材料が出たら更新するか**を明記

### 7-4. 打ち手の優先順位（役割別）
- 情シス／調達・法務／AI ガバナンス／経営・投資 など、**役割ごとに一手目・二手目**
- 「自社の文脈に当てはめて読み替えてください」の一言を添える

### 7-5. 次に見る指標（1〜2週間〜3か月）
- 他通信社の独立確認・公聴会・契約公示・公式追加発信・訴訟進行など、**読者側でも確認できる**観測点
- ここを入れることで、**後日読み返しても価値が残る**アーカイブ性を担保する
```

### 筆者メモで外してはいけないこと

- **取材の有無と免責**の再掲（無料冒頭の免責と同じ趣旨を 1 行）
- **報道・一次事実そのものは有料に移さない**（誤読防止の最低線は本編に残す）
- **「結論を有料で明かす」構造にしない**（有料は**判断の深さ**で売る）

## 補助稿のテンプレ（見出し骨子）

```markdown
# 「ITニュース・補助」｜〈本編と対になる短い題〉

## 本稿の位置づけ
（本編へのリンク・本編が担うこと／補助が担うこと）

## おすすめの読み順
（初見／再読／有料だけ、など）

## 1. 用語ミニ辞典
## 2. 年表（必要なときだけ）
## 3. 報道と公式のレイヤー
## 4. 本編に入る前のチェックリスト（任意）
```

## 品質チェック

**無料側**

- [ ] 補助が「本編の焼き直し長文」になっていない（**足場**に徹している）
- [ ] 本編冒頭に**読み順＋補助リンク**がある
- [ ] 本編の時間軸整理が**観察レーン**に収まっている（順序づけ・比率・役割別打ち手を有料から漏らしていない）
- [ ] **結論の人質**をしていない（「詳しくは有料で」で無料を閉じていない）
- [ ] 相互リンクの相対パスが切れていない

**有料側（筆者メモ）**

- [ ] 見出し直後（または直前の無料表示領域）に**「引き受けること」リード**がある
- [ ] 7-1〜7-5 のいずれかが**中身空**になっていない（空なら節ごと削る）
- [ ] 7-1 で**比率感が「確度」ではなく「注視順」**として書かれている
- [ ] 7-3（反論・限界）が**セルフ反証**になっている（補助の延長になっていない）
- [ ] 7-5（次に見る指標）が**読者側でも確認可能な粒度**で書かれている
- [ ] 筆者メモが「補助の延長」や「本編の続き」だけになっていない
