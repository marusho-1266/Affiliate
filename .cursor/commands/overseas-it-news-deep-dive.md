# 海外・日本 ITニュース・深掘り報告（note 向けブリーフ）

`news-log.md`（**海外・日本**の速報を累積）の**1行・1トピック**、またはユーザーが指定した**URL・件名**について、**note 執筆前の報告資料**（背景 / 目的 / 効果）を日本語でまとめる。実行時は必ずリポジトリ内スキル **`.cursor/skills/note-deep-dive-report/SKILL.md`** に従う。

## いつ使うか

- `reports/overseas-it-news/news-log.md` の箇条書きを **@ 参照**し「本件について深堀」と依頼されたとき
- 速報ログに載っていないが、**単発トピック**を同じ形式で深掘りしたいとき

## 入力の取り方

1. **ユーザーがファイル＋行・範囲を添付した場合** — そのトピックを対象とする（出典URLがログにあればそれを主出典の起点にする）。
2. **URL のみ** — そのURLを一次・準一次として開き、関連する公式・当局を追加検索する。
3. **件名のみ** — Web検索で**公式・当局を優先**し、時系列と主出典を確定してから執筆する。

## 深掘りモード（スキルに準拠）

| モード | 条件 | 手順 |
|--------|------|------|
| **ライト** | 単一トピック、主出典が1〜2本で足りる、規制・資本・CVEが主役でない | 親エージェントが Web 検索・取得で事実を集約し、下記テンプレに収める。 |
| **フル** | 規制・市場・技術・前史が同時に絡む、高リスク（侵害・訴訟・大型資金）、ユーザーが「フル」「並列」「出典以外も」と指示 | スキル **`.cursor/skills/note-deep-dive-report/subagents/README.md`** の対応表に従い、**Task** でレーン **A（一次・公式）は必須**、必要に応じて B〜E を**同一ターンで並列**起動。返却を **A→B/C/D→E** のマージ順で統合し、矛盾は一次優先で注記。 |

Task が使えない場合は、スキル記載のとおり親が各レーンのプロンプトを**自ら順に実行**して同じ見出しでマージする。

## 出力（ファイル）

| 項目 | 内容 |
|------|------|
| **既定の保存先** | `reports/note-briefs/YYYY-MM-DD-<英語スラッグ>.md` |
| **スラッグ** | トピックを表す短い英単語・ハイフン区切り（例: `openai-122b-funding`, `introducing-gpt-5-4`） |
| **ユーザーがパスを指定した場合** | そのパスに保存（拡張子 `.md`） |

`news-log.md` や `seen-urls.json` への**自動追記は不要**（本コマンドはブリーフ専用）。ユーザーが「ログにも反映」と明示した場合のみ、`news-log.md` の運用ルールに従って追記する。

## 報告資料の必須構成

スキル内の **「報告資料テンプレート」** をそのまま使う。特に次を満たすこと。

- **1. 背景** — 「なぜ今」を**事実**と**推測**に分ける。根拠のない動機の断定をしない。
- **2. 目的** — 想定読者ごとに**行動**が具体的に書いてある表（または同等の構造）。
- **3. 効果** — **短期（0〜3か月）／中期（3か月〜1年）／長期（1年以上）**の定義を明記し、各段で**不確実性**と**効果が出ない条件**を書く。
- **4. note 執筆メモ** — フック案、要検証点、免責（該当時）。
- **5. 参照一覧** — 一次情報を優先し、二次のみの場合はその旨。

数値・日付は**出典付き**のみ。メディア間で食い違う数字は**一次で照合**し、資料に注意書きする。

## チャットでの返答

- トピックの**要約（数文）**
- **保存したファイルのフルパス**
- フル調査をした場合は、どのレーン（A〜E）を使ったか**一行**でよいので明示

## ツール

- **Web 検索**と**ページ取得**で公式・当局を優先する。
- フルモードでは **Task**（`subagent_type`: `explore` または `generalPurpose`、`readonly: true` 推奨）でサブエージェントを並列起動する。
- スキル本文・`reference.md`・各 `subagents/*.md` の **「Task 用プロンプト」** を必要に応じて読む。
