# note 記事作成（ブリーフ指定）

ユーザーが **`reports/note-briefs/*.md` の報告資料を @ 参照**して指定したとき、その内容を底本に **note 掲載用の本稿 Markdown** を書く。実行時はリポジトリ内スキル **`.cursor/skills/note-article-from-brief/SKILL.md`** に従う。

**本稿禁止:** note 記事本文に、ブリーフ由来・`reports/note-briefs/` パス・「執筆前ブリーフ」等の**作成プロセスや内部ファイルへの言及を書かない**（スキル「本稿に書かないこと」参照）。

**平易さ:** 専門語は初出で短く説明し、略語が続く記事ではリード直後に **「この記事での用語」** 小表（3〜8語）を置く。詳細はスキル **「平易さ（用語・アクセス）」**。

## いつ使うか

- `2026-04-15-openai-gpt-5-4-cyber-tac.md` のように、**既存の note ブリーフ**から本稿を一気に作りたいとき
- 深掘り調査でブリーフを書いた**直後**に、`note/20######/` へ落としたいとき

## 入力の取り方

1. **ブリーフファイルパス（必須）** — ユーザーが `@reports/note-briefs/...` で添付するか、パスを明示する。
2. **任意** — 保存ファイル名・シリーズ名（既定はスキル記載）、前提としてリンクしたい既存 `note/` 稿、一次確認の有無。

## 出力

| 項目 | 内容 |
|------|------|
| **既定の保存先** | ブリーフの **調査日** に対応する `note/YYYYMMDD/YYYY-MM-DD-<スラッグ>.md`（詳細はスキル） |
| **ユーザーがパスを指定した場合** | そのパスに保存 |

ブリーフ本文ファイル（`reports/note-briefs/`）は**原則として書き換えない**。差分が出た一次確認結果は **本稿側**に反映する。

## チャットでの返答

- 記事の**要旨（数文）**
- **保存したファイルのリポジトリ相対パス**
- 実施した場合のみ、**一次確認**で触れた URL を一行程度で明示

## ツール

- **Read** でブリーフ全文を読む。
- スキルどおり**短い一次確認**をするときのみ **Web 検索／ページ取得**を使う。
- 完成稿を **Write** で `note/` に保存する。

## 有料化レディで書くとき

**筆者メモ（note 本番では有料ブロック想定）**を同時に置きたい場合は、本編末尾「主な参照」の**直後**に `## 〈番号〉. 筆者メモ（…）` を追加する。**振り分け規則（観察は無料／判断は筆者メモ）と 7-1〜7-5 の雛形**は `.cursor/skills/note-trio-pack/SKILL.md` を参照（本コマンド・本スキルは正本にしない）。既存本編の**有料化レディ再編だけ**したいときは `.cursor/commands/note-trio-pack.md` を使う。

## 関連

- ブリーフをまだ持っていない場合: `.cursor/commands/overseas-it-news-deep-dive.md` と `.cursor/skills/note-deep-dive-report/SKILL.md`
- 三層（本編＋補助＋筆者メモ）: `.cursor/commands/note-trio-pack.md` と `.cursor/skills/note-trio-pack/SKILL.md`
- テンプレ: `note/ニュースnoteテンプレ.md`（無料＋筆者メモの並び）
