# note／記事からラジオ台本（Voicemaker・ポッドキャスト向け）

ユーザーが **`@` で指定した Markdown 記事**（または要点メモ）をもとに、**読み上げ用ラジオ台本**を1ファイルで出力する。実行時は必ずリポジトリ内スキル **`.cursor/skills/radio-script-from-note/SKILL.md`** に従う（構造・文体・TTS 規則・品質チェックはスキル本文が正）。

## いつ使うか

- note 記事やブログ稿を **ポッドキャスト／Voicemaker／音声配信**向けに変換したい
- **〔転換・読み上げ用〕** 付きの運用ルールどおりの台本が欲しい
- **Markdown 装飾を読み上げに載せたくない**（TTS でノイズになる）前提でプレーン台本が欲しい

## エージェントが最初にすること（必須）

1. **`.cursor/skills/radio-script-from-note/SKILL.md` を読む**（未読のまま台本を書かない）。
2. ユーザーが **`@` で `.md` を指定**していればそのファイルを入力とする。指定がなければ、会話内の本文・要点メモを入力としてよい（スキルの「作業フロー」に従う）。

## 入力の取り方

| パターン | 扱い |
|----------|------|
| **記事ファイル＋ `@` 参照** | その `.md` を優先して読み、メタ・一次ソース・免責を拾う。**有料パート**として **`##` 見出しに「筆者メモ」が付いた行からファイル末尾までは台本に含めない**（詳細はスキル「入力の切り取り」）。 |
| **尺・パート数・一人称／第三者**など | ユーザーの**そのセッション内の明示**はスキルより優先（スキル末尾「ユーザー指示の優先」） |

## 出力（ファイル）

| 項目 | 内容 |
|------|------|
| **既定の保存先** | 元記事と**同じディレクトリ** |
| **ファイル名の例** | `〈元記事のベース名〉｜ラジオ台本.md` |
| **ユーザーがパスを指定した場合** | そのパスに保存 |

台本の**必須ブロック**（タイトル行・メタ・読み上げのしかた・オープニング・本編パート・クロージング・制作メモ・配信説明文）、**TTS 規則**（英二語連結、文末「。」の改行、比較の「ほう」など）、**品質チェック**はいずれもスキルに従う。**筆者メモ以下（有料想定）を台本に含めない**こともスキルどおりとする。

**放送本編の語彙**: 読み上げ本文では **「本稿」**（本稿では／本稿の論点 等）を使わない。記事メタを避け、**「今日の話では」「今日の話の焦点」「今日の話は」** などに言い換える。時間軸の定義など **「この回の置き方」** を示す文では **「この話では」** も使い分け可（詳細はスキル「文体・メタ表現の回避」）。

## チャットでの返答

- **保存した台本ファイルのパス**（フルパスまたはリポジトリからの相対パス）
- **想定尺**がスキルどおり入っているか一言
- ユーザーが求めていれば、**筆者＝ナレーター同一**か**第三者**かの確認

## ツール

- 入力がリポジトリ内ファイルなら **Read** で全文を読む。
- スキルに載っている **参考例**（`note/` 配下の既存ラジオ台本）を必要に応じて読み比べてよい。
